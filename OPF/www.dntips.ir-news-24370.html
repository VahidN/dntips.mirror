‫کتابخانه UI جدید htmxRazor برای ASP.NET Core معرفی شد


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۸ ۰۰:۵۲
آدرس: www.dntips.ir
توسعه‌دهندگان ASP.NET Core برای ایجاد رابط‌های کاربری سرور-رندر شده، اغلب مجبور به استفاده از کتابخانه‌های جاوااسکریپتی مانند React، Vue یا Angular هستند. این در حالی است که هدف اصلی آن‌ها استفاده از قابلیت‌های سرور-رندرینگ بوده است. پروژه htmxRazor به عنوان یک کتابخانه UI متن‌باز مبتنی بر Tag Helpers در ASP.NET Core، برای رفع این مشکل طراحی شده است. این کتابخانه شامل ۷۲ کامپوننت آماده تولید در ۱۰ دسته‌بندی مختلف از جمله دکمه‌ها، کنترل‌های فرم، دیالوگ‌ها، تب‌ها، ناوبری، اسلایدرها، تجسم داده‌ها و نشانگرهای بازخورد است. هر کامپوننت، HTML تمیز و معنایی را در سرور رندر می‌کند و از ویژگی‌های htmx به عنوان خواص اصلی خود پشتیبانی می‌کند. این بدان معناست که htmxRazor به عنوان یک کتابخانه کامپوننتی طراحی شده که به طور خاص با htmx سازگار است. تنظیمات htmxRazor تنها با دو خط کد در فایل Program.cs انجام می‌شود: افزودن سرویس‌ها و فعال‌سازی آن در خط لوله درخواست. برای استفاده از کامپوننت‌ها، کافی است Tag Helpers مربوطه را در فایل _ViewImports.cshtml ثبت کنید. این فرآیند بدون نیاز به پیکربندی webpack، نصب npm یا استفاده از bundler انجام می‌شود و تنها با یک بسته NuGet قابل پیاده‌سازی است. htmxRazor به دنبال هدف قرار دادن توسعه‌دهندگانی است که از رندرینگ سرور لذت می‌برند، از htmx استفاده می‌کنند و به کامپوننت‌های واقعی بدون سربار اضافی نیاز دارند. این کتابخانه با ارائه ۷۲ کامپوننت، نیازهای واقعی برنامه‌ها را در ۱۰ دسته‌بندی مختلف پوشش می‌دهد و بدون نیاز به runtime سمت کلاینت، امکان توسعه رابط‌های کاربری سرور-رندر شده را فراهم می‌کند.
