‫معرفی SQL Server 2025: پایگاه داده آماده هوش مصنوعی برای سازمان‌ها


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۲/۳۱ ۰۸:۲۰
آدرس: www.dntips.ir
مایکروسافت پیش‌نمایش عمومی SQL Server 2025 را معرفی کرده است. این نسخه که به عنوان "پایگاه داده آماده هوش مصنوعی (AI-ready enterprise database)" معرفی شده، با هدف توانمندسازی سازمان‌ها برای توسعه برنامه‌های کاربردی هوش مصنوعی مدرن و ایمن بر پایه داده‌های خود طراحی شده است. SQL Server 2025 ویژگی‌های هوش مصنوعی را به صورت داخلی و قابل توسعه ارائه می‌دهد، بهره‌وری توسعه‌دهنده را افزایش می‌دهد و یکپارچگی بی‌نقصی با سرویس‌های ابری مایکروسافت از جمله Microsoft Azure و Microsoft Fabric فراهم می‌آورد. همه این قابلیت‌ها در هسته موتور SQL Server و با استفاده از زبان آشنای T-SQL قابل دسترسی هستند.

یکپارچه‌سازی هوش مصنوعی در هسته پایگاه داده
یکی از هیجان‌انگیزترین قابلیت‌های جدید SQL Server 2025، ادغام مستقیم هوش مصنوعی در موتور پایگاه داده (integration of AI directly into the database engine) است که امکان جستجوی هوشمندانه‌تر را فراهم می‌کند. این نسخه با قابلیت‌های جستجوی برداری (vector search) داخلی، به کاربران اجازه می‌دهد تا جستجوهای معنایی (semantic searches) را روی داده‌های خود انجام دهند. این قابلیت علاوه بر جستجوی متنی کامل (full text search) و فیلترینگ موجود در SQL Server، امکان یافتن تطابق‌ها بر اساس شباهت را نیز فراهم می‌آورد. این ویژگی، موارد استفاده جدیدی مانند کشف ارتباطات عمیق‌تر در مجموعه داده‌های بزرگ و ارائه تجربه مکالمه طبیعی (natural conversational experience) در سیستم‌های سازمانی مختلف را ممکن می‌سازد.
SQL Server 2025 همچنین مدیریت مدل (model management) را با ساخت تعاریف مدل به صورت مستقیم در T-SQL بهبود بخشیده است. این امر، یکپارچگی بی‌نقصی را با سرویس‌های محبوب هوش مصنوعی مانند Azure AI Foundry، Azure OpenAI Service، OpenAI و Ollama فراهم می‌کند. مدل‌ها از طریق رابط‌های برنامه‌نویسی کاربردی REST (REST APIs) قابل دسترسی هستند که به توسعه‌دهندگان اجازه می‌دهد هر مدلی را به صورت ایمن و ایزوله از موتور SQL Server، در هر محیطی از "زمین تا ابر" (ground to cloud) مستقر کنند. این رویکرد، تغییر مدل‌ها را بدون نیاز به تغییر کد، تسهیل می‌نماید و به توسعه‌دهندگان در انتخاب بهترین مدل برای موارد استفاده خود کمک می‌کند.
علاوه بر این، SQL Server 2025 سایر بلوک‌های سازنده ضروری برای توسعه هوش مصنوعی (AI development) و الگوهای بازیابی تقویت‌شده با تولید (retrieval-augmented generation – RAG) عملیاتی را که توسط عامل‌های هوش مصنوعی (AI agents) پشتیبانی می‌شوند، ارائه می‌دهد. این موارد شامل تولید جاسازی برداری (vector embedding generation) و تقسیم‌بندی متن (text chunking) است که به صورت داخلی در T-SQL قرار گرفته‌اند و از Disk Approximate Nearest Neighbor (DiskANN) به عنوان یک شاخص برداری (vector index) برای نتایج سریع‌تر، کارآمدتر از نظر منابع و دقیق‌تر استفاده می‌کنند. همچنین، این نسخه یکپارچگی بی‌نقصی را با چارچوب‌های محبوب هوش مصنوعی مانند LangChain، Semantic Kernel و Entity Framework Core ارائه می‌دهد.

تحول در تجربه توسعه‌دهندگان SQL
SQL Server 2025 با مجموعه گسترده‌ای از بهبودهای ویژگی، فرآیند توسعه را به طور قابل توجهی ساده، پیچیدگی کد را کاهش و بهره‌وری توسعه‌دهنده را بهبود می‌بخشد. مایکروسافت این نسخه را "مهم‌ترین انتشار برای توسعه‌دهندگان SQL از زمان معرفی SQL Server 2016" می‌داند.
غنی‌سازی داده (Data Enrichment) یکی از نقاط تمرکز اصلی در این نسخه است. SQL Server 2025 پشتیبانی بومی JSON (native JSON support) را ارائه می‌دهد که به توسعه‌دهندگان امکان پردازش مستقیم اسناد JSON را می‌دهد. این قابلیت، همراه با فعال‌سازی REST APIs و عبارات منظم (Regular Expressions – RegEx)، به توسعه‌دهندگان اجازه می‌دهد تا مجموعه داده‌های خود را با منابع داده خارجی غنی‌سازی، اعتبارسنجی و دستکاری کنند. این ویژگی‌ها امکان ساخت برنامه‌های کاربردی سازمانی پویاتر با قابلیت‌های غنی‌تر و عملکرد بهبود یافته را فراهم می‌آورد.
SQL Server 2025 همچنین سناریوی ساخت برنامه‌های کاربردی بلادرنگ و رویدادمحور (real-time, event-driven applications) را با SQL Server امکان‌پذیر می‌سازد. قابلیت Change Event Streaming به کاربران اجازه می‌دهد تا تغییرات گزارش تراکنش (transaction log) را به عنوان رویدادهایی به صورت مستقیم از SQL Server به Microsoft Azure Event Hubs مصرف کنند. این یک روش جدید برای کاهش برخی از مسائل مربوط به سربار ورودی/خروجی (I/O overhead) تصویربرداری تغییر داده‌ها (Change Data Capture – CDC) را فراهم می‌آورد و امکانات جدیدی مانند توسعه برنامه‌های کاربردی بلادرنگ و رویدادمحور که توسط عامل‌های هوش مصنوعی پشتیبانی می‌شوند را باز می‌کند.
در بخش زبان و ابزارها نیز پیشرفت‌های قابل توجهی وجود دارد. مایکروسافت از پیش‌نمایش درایور پایتون متن‌باز جدید (new open-source Python driver) برای SQL Server خبر داده است. این درایور که از پایه ساخته شده، یک راه‌حل قدرتمند، کارآمد و کاملاً متن‌باز را برای توسعه‌دهندگان پایتون جهت اتصال به SQL Server ارائه می‌دهد که به سادگی با دستور pip install قابل نصب است. علاوه بر این، افزونه MSSQL برای Visual Studio Code با GitHub Copilot یکپارچه شده است. این قابلیت که در حال حاضر در پیش‌نمایش قرار دارد، هوش مصنوعی را مستقیماً به جریان کار توسعه‌دهنده می‌آورد. با آگاهی GitHub Copilot از اتصال پایگاه داده SQL Server، توسعه‌دهندگان می‌توانند SQL و مهاجرت‌های نقشه‌برداری رابطه‌ای شیء (object-relational mapping – ORM migrations) را تولید کنند، طرح‌واره‌ها (schemas) را بررسی کنند، پرس‌وجوها را با پیشنهادات هوشمند بهینه‌سازی کنند و تعاملات پایگاه داده را ساده‌تر سازند، همه این‌ها در محیط Visual Studio Code انجام می‌شود.
در نهایت، مایکروسافت یک ویرایش توسعه‌دهنده استاندارد جدید (new Standard Developer edition) را معرفی می‌کند تا ویرایش توسعه‌دهنده موجود خود را که شامل تمام ویژگی‌های Enterprise است، تکمیل کند. این نسخه جدید به توسعه‌دهندگان راهی را برای ساخت و آزمایش در برابر سطح ویژگی‌های نسخه استاندارد (Standard edition surface area) بدون تحمیل هزینه‌های مجوز ارائه می‌دهد.

امنیت، عملکرد و در دسترس بودن در سطح سازمانی
SQL Server 2025 بر سابقه SQL Server به عنوان یک رهبر صنعتی در زمینه امنیت، عملکرد و در دسترس بودن پایگاه داده تکیه دارد. این نسخه، میراث SQL Server در امنیت درجه یک را با ترکیب روش‌های مدرن شناسایی و رمزنگاری (modern identity and encryption practices) ادامه می‌دهد. پشتیبانی از هویت‌های مدیریت شده Microsoft Entra (Microsoft Entra managed identities)، مدیریت اعتبارات را بهبود می‌بخشد و آسیب‌پذیری‌های احتمالی را کاهش می‌دهد.
مایکروسافت قفل‌گذاری بهینه (Optimized Locking) را به SQL Server آورده است تا مصرف حافظه قفل را کاهش داده و مسدود شدن (blocking) برای تراکنش‌های هم‌زمان را از طریق قفل‌گذاری شناسه تراکنش (Transaction ID Locking – TID Locking) و قفل پس از صلاحیت (Lock After Qualification – LAQ) به حداقل برساند. این قابلیت به مشتریان امکان می‌دهد تا زمان آپدیت (uptime) را افزایش داده و هم‌زمانی و مقیاس‌پذیری (scale) را برای برنامه‌های کاربردی SQL Server بهبود بخشند.
SQL Server 2025 بیش از 50 بهبود در موتور پایگاه داده دارد، از جمله بهبودهای کلیدی برای در دسترس بودن بالا و بازیابی فاجعه (High Availability and Disaster Recovery – HADR) که همه بر اساس بازخورد مشتریان است. این نسخه جدید، بهبودهایی در عملکرد برنامه‌ها بدون نیاز به تغییر کد را از طریق پردازش پرس‌وجوی هوشمند (Intelligent Query Processing – IQP) و بهبودهای ذخیره‌سازی ستونی (columnstore)، بهبودهایی برای پردازش پرس‌وجوها و فعال‌سازی Query Store برای ثانویه‌های خواندنی (readable secondaries) به ارمغان می‌آورد.
