‫استفاده از Redis در ASP.NET با .NET


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۳۰ ۱۰:۳۱
آدرس: www.dntips.ir
این مقاله به بررسی نحوه استفاده از Redis در برنامه‌های ASP.NET با استفاده از .NET می‌پردازد. Redis یک پایگاه داده کلید-مقدار سریع و در حافظه است که اغلب برای ذخیره‌سازی کش، مدیریت جلسات و سایر وظایف با کارایی بالا استفاده می‌شود. برای شروع، باید بسته‌ی NuGet StackExchange.Redis را از طریق خط فرمان dotnet add package StackExchange.Redis نصب کنید. سپس، رشته‌ی اتصال Redis را در فایل appsettings.json پیکربندی کنید. به عنوان مثال، "ConnectionStrings": { "Redis": "localhost:6379" }. در مرحله‌ی بعد، باید رشته‌ی اتصال را در Program.cs پیکربندی کنید تا یک نمونه‌ی IConnectionMultiplexer را به عنوان یک سرویس تک‌ساده (Singleton) ثبت کنید. این کار به شما امکان می‌دهد به راحتی به Redis از طریق تزریق وابستگی (DI) دسترسی داشته باشید. برای استفاده از Redis در کد، می‌توانید IConnectionMultiplexer را به کلاس‌های خود تزریق کنید. به عنوان مثال، در یک میانی‌لایه (Middleware) برای اعتبارسنجی توکن‌های JWT، می‌توانید از _redis.GetDatabase().StringGetAsync برای بازیابی نسخه‌ی JWT استفاده کنید. در تست‌های واحد (Unit Tests)، می‌توانید از Moq و AutoFixture برای mock کردن IConnectionMultiplexer و شبیه‌سازی رفتار Redis استفاده کنید. هنگام mock کردن GetDatabase، باید پارامترهای پیش‌فرض را در نظر بگیرید، زیرا GetDatabase در C# از مقادیر پیش‌فرض برای پارامترها استفاده می‌کند.
