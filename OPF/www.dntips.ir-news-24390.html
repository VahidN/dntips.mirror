‫راه اندازی توسعه محلی: ابزارها، اشکال زدایی و بارگذاری مجدد داغ در Azure Functions


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۸ ۱۱:۳۰
آدرس: www.dntips.ir
در توسعه محلی Azure Functions با استفاده از .NET، اشکال زدایی می‌تواند چالش برانگیز باشد. دلیل این امر این است که هنگام اشکال زدایی یک تابع isolated worker، دو فرآیند جداگانه اجرا می‌شوند و اشکال‌زدا به فرآیند host (func.exe) متصل می‌شود، نه فرآیند worker (dotnet.exe) که کد شما در آن اجرا می‌شود. فایل local.settings.json نیز می‌تواند باعث سردرگمی شود، زیرا متغیرهای محیطی را تزریق می‌کند. قرار دادن یک رشته اتصال در بخش اشتباه می‌تواند باعث از کار افتادن bindings شود. همچنین، Azurite یک تله مرتبط است؛ triggerهای timer، queue و blob همگی به صورت داخلی از Azure Storage استفاده می‌کنند، بنابراین emulator باید در حال اجرا باشد تا این انواع triggerها حتی قبل از اینکه کد تابع شما به هیچ storageای دسترسی پیدا کند، مقداردهی اولیه شوند. برای اجرای هر Azure Function به صورت محلی، سه مورد باید نصب و در حال کار باشند: .NET 10 SDK، Azure Functions Core Tools v4 و Azurite. Azurite فضای ذخیره‌سازی Azure Blob، Queue و Table را به صورت محلی شبیه‌سازی می‌کند و برای هر تابع غیر-HTTP ضروری است. این راهنما، راه اندازی کامل محلی برای Azure Functions با .NET 10 و مدل isolated worker را پوشش می‌دهد. مواردی مانند deployment و Application Insights در این راهنما وجود ندارند و در قسمت‌های بعدی بررسی خواهند شد.
