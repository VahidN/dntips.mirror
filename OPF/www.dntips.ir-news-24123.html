‫شروع کار با زمان‌بندی در .NET با استفاده از Hangfire


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۳۰ ۰۹:۱۷
آدرس: www.dntips.ir
این مقاله به بررسی کتابخانه Hangfire برای زمان‌بندی وظایف در برنامه‌های .NET می‌پردازد. Hangfire یک ابزار قدرتمند برای زمان‌بندی و اجرای وظایف پس‌زمینه در برنامه‌های .NET است و به شما امکان می‌دهد وظایف را به صورت دوره‌ای یا در زمان‌های مشخص اجرا کنید. این کتابخانه از پایگاه داده یا سرویس کش مانند Redis برای ذخیره و مدیریت وظایف استفاده می‌کند. برای شروع، باید بسته‌های Nuget مورد نیاز، شامل Hangfire.Core، Hangfire.AspNetCore و Hangfire.SqlServer را نصب کنید. همچنین، یک پایگاه داده خالی در SQL Server (یا SQLExpress) ایجاد کنید و یک رشته اتصال (connection string) در فایل appsettings.json تنظیم کنید تا Hangfire بتواند به پایگاه داده متصل شود. در مرحله بعد، سرویس Hangfire را در برنامه خود راه‌اندازی کنید. این کار با استفاده از کد در Program.cs انجام می‌شود که Hangfire را پیکربندی کرده و یک داشبورد برای نظارت بر وظایف در حال اجرا فراهم می‌کند. تنظیمات سازگاری نسخه و استفاده از Serializer مناسب نیز در این مرحله انجام می‌شود. Hangfire از جداول مختلفی در پایگاه داده برای ذخیره اطلاعات مربوط به وظایف، سرورها و تنظیمات استفاده می‌کند. این جداول شامل AggregatedCounter، Counter، Hash، Job، JobParameter، JobQueue، List، Schema، Server و Set هستند. هر جدول اطلاعات خاصی را در مورد وضعیت و عملکرد وظایف ذخیره می‌کند. در نهایت، با استفاده از داشبورد Hangfire، می‌توانید وظایف زمان‌بندی شده را مشاهده، مدیریت و عیب‌یابی کنید. Hangfire یک راه حل ساده و کارآمد برای زمان‌بندی وظایف در برنامه‌های .NET فراهم می‌کند و به توسعه‌دهندگان کمک می‌کند تا وظایف پس‌زمینه را به راحتی مدیریت کنند.
