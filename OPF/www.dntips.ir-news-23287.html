‫معماری یک برنامه تمام‌عیار با Flutter و Dart بر روی Cloud Run


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۰ ۰۷:۳۲
آدرس: www.dntips.ir
این مقاله به بررسی نحوه ساخت و استقرار یک برنامه تمام‌عیار با استفاده از Flutter و Dart بر روی Google Cloud Run می‌پردازد. Flutter یک فریم‌ورک برای ساخت برنامه‌های موبایل، وب و دسکتاپ از یک کدبیس واحد است و زبان Dart که قدرت Flutter را تامین می‌کند، برای توسعه بک‌اند نیز مناسب است. استفاده از Dart هم برای فرانت‌اند و هم برای بک‌اند، یک معماری تمام‌عیار را امکان‌پذیر می‌سازد که مزایایی مانند ساده‌سازی کد، اشتراک‌گذاری منطق بین کلاینت و سرور و استقرار کل استک در یک کانتینر واحد را به همراه دارد. Cloud Run به عنوان یک پلتفرم بدون سرور و مدیریت‌شده، استقرار را ساده می‌کند. این پلتفرم زیرساخت و مقیاس‌بندی را برای شما مدیریت می‌کند و شما فقط باید کانتینر را ارائه دهید. در نتیجه، یک URL امن HTTPS برای کل استک شما در عرض چند دقیقه به دست می‌آورید. در این مقاله، یک برنامه To-Do به عنوان یک مثال عملی برای نشان دادن این فرآیند استفاده شده است و تمام کدها در مخزن Flutter Cloud Run قابل دسترسی هستند. در بخش فرانت‌اند، از Flutter و ویجت‌ها برای ساخت رابط کاربری استفاده می‌شود. منطق رندر کردن یک وظیفه واحد در یک ویجت TodoTile جدا شده است. این ویجت، مدل داده TodoItem را مستقیماً از یک پکیج مشترک که در کل استک استفاده می‌شود، وارد می‌کند. این امر تضمین می‌کند که در صورت تغییر تعریف TodoItem، کامپایلر شما را در مورد به‌روزرسانی ویجت‌های UI مطلع می‌کند. در بخش بک‌اند، از پکیج Shelf در Dart استفاده شده است. Shelf به توسعه‌دهندگان اجازه می‌دهد تا با انعطاف‌پذیری بیشتری با بک‌اند کار کنند و کنترل بیشتری بر روی استقرار و مدیریت کد داشته باشند. این رویکرد امکان استقرار در محیط‌های زیرساختی مختلف و همچنین استفاده مجدد از کد بین کلاینت و سرور را فراهم می‌کند.
