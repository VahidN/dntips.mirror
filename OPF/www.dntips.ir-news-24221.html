‫اتصال عامل‌ها به ابزارهای OpenAPI در Microsoft Foundry


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۵ ۰۰:۳۳
آدرس: www.dntips.ir
مایکروسافت Foundry به شما امکان می‌دهد عامل‌های خود را به APIهای خارجی با استفاده از مشخصات OpenAPI 3.0 متصل کنید. این قابلیت به عامل‌ها اجازه می‌دهد تا سرویس‌های خارجی را فراخوانی کرده، داده‌های بلادرنگ را بازیابی کنند و قابلیت‌های خود را فراتر از توابع داخلی گسترش دهند. مشخصات OpenAPI یک روش استاندارد برای توصیف APIهای HTTP ارائه می‌دهد که به شما امکان می‌دهد سرویس‌های موجود را با عامل‌های خود یکپارچه کنید. Microsoft Foundry از سه روش احراز هویت پشتیبانی می‌کند: ناشناس، کلید API و هویت مدیریت شده. برای انتخاب روش احراز هویت مناسب، به صفحه "Choose an authentication method" مراجعه کنید. پشتیبانی از SDKهای مختلف برای زبان‌های برنامه‌نویسی مختلف ارائه شده است، از جمله Python، C#، JavaScript و Java. برای Java، از پکیج com.azure:azure-ai-agents استفاده کنید. در تنظیمات محیطی، AZURE_AI_PROJECT_ENDPOINT باید به آدرس نقطه پایانی پروژه Foundry شما اشاره داشته باشد، نه نقطه پایانی سرویس OpenAPI خارجی. برای استفاده از این قابلیت، پیش‌نیازهایی وجود دارد، از جمله داشتن اشتراک Azure با مجوزهای مناسب، نقش Contributor یا Owner در پروژه Foundry، یک پروژه Foundry با endpoint پیکربندی شده و یک مدل هوش مصنوعی مستقر در پروژه. همچنین، یک محیط عامل پایه یا استاندارد مورد نیاز است و باید SDK مربوط به زبان برنامه‌نویسی مورد نظر نصب شده باشد. محدودیت‌هایی نیز در این مورد وجود دارد. مشخصات OpenAPI شما باید شامل operationId برای هر عملیات باشد و operationId فقط می‌تواند شامل حروف، - و _ باشد. همچنین، از انواع محتوای application/json و application/json-patch+json پشتیبانی می‌شود. برای احراز هویت با کلید API، از یک طرح امنیتی کلید API در هر ابزار OpenAPI استفاده کنید.
