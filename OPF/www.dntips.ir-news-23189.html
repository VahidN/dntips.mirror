‫GitHub Copilot SDK در پیش‌نمایش فنی قرار گرفت


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۰۹ ۱۲:۵۴
آدرس: www.dntips.ir
GitHub Copilot SDK وارد مرحله پیش‌نمایش فنی شده است. این SDK به توسعه‌دهندگان امکان می‌دهد لایه هماهنگی را برای گردش‌کارهای مبتنی بر عامل (agentic workflows) ایجاد کنند و وظایفی مانند برنامه‌ریزی، فراخوانی ابزارها/MCP و مدیریت وضعیت و استریم را انجام دهند. Copilot یک سیستم کمک‌کننده کد مبتنی بر هوش مصنوعی از GitHub است که در ابتدا بر اساس Codex از OpenAI بود، اما اکنون از مدل‌های چندگانه پشتیبانی می‌کند و به توسعه‌دهندگان این امکان را می‌دهد تا از بین مدل‌های مختلف انتخاب کنند. SDK جدید، موتور پشت Copilot CLI را در اختیار توسعه‌دهندگان قرار می‌دهد. CLI کمک‌کننده کد با هوش مصنوعی را مستقیماً در خط فرمان ارائه می‌دهد و به توسعه‌دهندگان امکان می‌دهد تا از طریق زبان طبیعی کد را بسازند، اشکال‌زدایی کنند و درک کنند. این SDK در حال حاضر از Python، TypeScript، Go و .NET پشتیبانی می‌کند. این ابزار می‌تواند زمینه را در طول تعاملات مدیریت کند، ابزارها و دستورات را هماهنگ کند، بین مدل‌ها مسیریابی کند، سرورهای MCP را ادغام کند و همچنین مجوزها، محدودیت‌های ایمنی و حالت‌های خرابی را مدیریت کند. توسعه‌دهندگان می‌توانند یک مدل را انتخاب کنند و در هر زمان بین مدل‌ها جابجا شوند. تیم توسعه‌دهندگان معتقد است که این SDK را می‌توان به عنوان یک پلتفرم اجرایی در نظر گرفت که به شما امکان می‌دهد از همان حلقه agentic که در پشت Copilot CLI قرار دارد، استفاده مجدد کنید، در حالی که GitHub مسئول احراز هویت، مدیریت مدل، سرورهای MCP، عامل‌های سفارشی و جلسات چت به همراه استریم است. به طور پیش‌فرض، SDK Copilot CLI را با ارسال معادل --allow-all به CLI اجرا می‌کند و به همه ابزارهای اصلی دسترسی می‌دهد. استفاده از این ابزار نیازمند اشتراک GitHub Copilot است و صورت‌حساب بر اساس همان مدل Copilot CLI است. همچنین از BYOK (Bring Your Own Key) پشتیبانی می‌کند، به طوری که توسعه‌دهندگان می‌توانند SDK را برای استفاده از کلیدهای API خود از ارائه دهندگان LLM پشتیبانی شده (مانند OpenAI، Azure، Anthropic) برای دسترسی به مدل‌ها از طریق آن ارائه دهندگان پیکربندی کنند.
