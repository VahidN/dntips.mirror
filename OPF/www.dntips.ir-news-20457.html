‫ویدئوهای «ابزارهای توسعه‌دهنده و دات‌نت | مایکروسافت بیلد ۲۰۲۵»

این گزیده شامل فهرستی از ویدئوها با عنوان "ابزارهای توسعه‌دهنده و دات‌نت | مایکروسافت بیلد ۲۰۲۵" از کانال Microsoft Developer است که شامل ۶۱ ویدئو می‌شود. محتوای اصلی این ویدئوها بر جنبه‌های مختلف توسعه از جمله هوش مصنوعی (AI)، دات‌نت (.NET)، سی‌شارپ (C#)، ویژوال استودیو (Visual Studio)، گیت‌هاب کوپایلوت (GitHub Copilot) و سایر فناوری‌های مایکروسافت تمرکز دارد.

نمای کلی لیست پخش

  • کانال: Microsoft Developer
  • تعداد ویدئوها: ۶۱

موضوعات کلیدی
ویدئوها طیف گسترده‌ای از موضوعات مرتبط با توسعه‌دهندگان را پوشش می‌دهند، با تأکید قوی بر موارد زیر:

هوش مصنوعی (AI) و کوپایلوت (Copilot):
  • توسعه هوش مصنوعی با VS Code برای وب
  • پرامپت‌های ضروری هوش مصنوعی برای توسعه‌دهندگان
  • افزایش بهره‌وری توسعه‌دهنده با گیت‌هاب کوپایلوت
  • ساخت، استقرار و استفاده از اولین سرور پروتکل زمینه مدل (MCP)
  • توسعه برنامه موبایل و دسکتاپ با هوش مصنوعی و .NET MAUI
  • ساخت نسل بعدی برنامه‌های هوش مصنوعی با .NET: مدل‌ها، داده‌ها، ایجنت‌ها و موارد دیگر
  • گیت‌هاب کوپایلوت برای NuGet: ساده‌سازی مدیریت بسته در ویژوال استودیو
  • افزایش پتانسیل توسعه‌دهنده با هوش مصنوعی و Dev Box
  • ساخت و استقرار سریع‌تر برنامه‌های هوش مصنوعی با کد کم و DevOps
  • درون‌مایه‌ها و جادوی گیت‌هاب کوپایلوت
  • ویژگی‌های برتر گیت‌هاب کوپایلوت که در ویژوال استودیو ۲۰۲۲ از دست داده‌اید
  • ارتقاء توسعه با .NET Aspire: ابری هوش مصنوعی و فراتر از آن
  • حالت ایجنت برای توسعه‌دهندگان "جدی"
  • بهبود کنترل‌های مدیریت ایجنت برای سازمان‌های آماده هوش مصنوعی
  • مدرن‌سازی ساده‌شده برنامه‌های جاوا با هوش مصنوعی
  • گسترش ایجنت کوپایلوت خود در Power Apps با Copilot Studio و SDKهای جدید
  • آینده مدرن‌سازی برنامه .NET با هوش مصنوعی
  • هوش مصنوعی مولد برای شرکت‌ها: برنامه‌ها و ایجنت‌های هوشمند با Dataverse و MCP
  • همکاری با تیمی از ایجنت‌ها برای ساخت راه‌حل‌های هوشمند
  • گیت‌هاب کوپایلوت برای شکاکان که هنوز فکر می‌کنند هوش مصنوعی بیش از حد ارزیابی شده است
  • شتاب‌بخشی به توسعه Azure با گیت‌هاب کوپایلوت، VS Code و هوش مصنوعی
  • پایتون و .NET: ساخت راه‌حل‌های هوش مصنوعی با نقاط قوت ترکیبی
  • استفاده از VS Code برای ساخت برنامه‌ها و ایجنت‌های هوش مصنوعی
  • رفع اشکال حرفه‌ای: بهبود کارایی با ویژوال استودیو و کوپایلوت
  • ساخت راه‌حل‌های مبتنی بر ایجنت با Power Platform و Copilot Studio
  • توسعه، ساخت و استقرار برنامه‌های LLM با استفاده از مدل‌های گیت‌هاب و Azure AI Foundry
  • حالت ایجنت در عمل: کدنویسی هوش مصنوعی با Vibe و جریان‌های مبتنی بر مشخصات
  • ایجنت بیدار می‌شود: توسعه مشترک با گیت‌هاب کوپایلوت
  • پیشگام شدن در آینده: DevOps مبتنی بر ایجنت با نیکول فورسگرن و جین کیم
  • چگونه توسعه‌دهندگان مایکروسافت از هوش مصنوعی در کدنویسی واقعی استفاده می‌کنند
  • بازنگری توسعه نرم‌افزار و DevOps با هوش مصنوعی ایجنتی
  • ساخت سریع نمونه اولیه، ساخت و استقرار برنامه‌های هوش مصنوعی با مدل‌های گیت‌هاب

NET. و توسعه سی‌شارپ (#C):
  • بدون نیاز به پروژه فقط سی‌شارپ با dotnet run app.cs
  • به‌روزرسانی رابط کاربری برنامه .NET MAUI در زمان طراحی
  • یادگیری نحوه نوشتن کد سی‌شارپ بهتر در ۱۵ دقیقه
  • نحوه قدرت‌دهی به سرویس‌های ایکس‌باکس در مقیاس بزرگ و افزایش بهره‌وری توسعه‌دهنده با C# و .NET
  • آینده توسعه وب با ASP.NET Core و Blazor
  • چه خبر در C#

ویژوال استودیو (Visual Studio) و VS Code:
  • توسعه هوش مصنوعی با VS Code برای وب
  • داستان ویژوال استودیو: چگونه نرم‌افزاری که میلیون‌ها توسعه‌دهنده دوست دارند را می‌سازیم
  • گیت‌هاب کوپایلوت برای NuGet: ساده‌سازی مدیریت بسته در ویژوال استودیو
  • ویژگی‌های برتر گیت‌هاب کوپایلوت که در ویژوال استودیو ۲۰۲۲ از دست داده‌اید
  • تسلط بر گیت در ویژوال استودیو
  • استفاده از VS Code برای ساخت برنامه‌ها و ایجنت‌های هوش مصنوعی
  • VS Code Live!
  • رفع اشکال حرفه‌ای: بهبود کارایی با ویژوال استودیو و کوپایلوت

گیت‌هاب (GitHub) و DevOps:
  • افزایش بهره‌وری توسعه‌دهنده با گیت‌هاب کوپایلوت
  • برنامه‌ریزی پروژه برای توسعه‌دهندگان با استفاده از گیت‌هاب
  • کار عملی با گیت‌هاب در شرکت
  • بزرگ‌تر کردن Azure DevOps و گیت‌هاب از مجموع اجزای آنها
  • نکات داخلی برای افزایش سرعت گردش کار گیت‌هاب شما

سایر موضوعات توسعه:
  • ۱۰۰ برابر سریع‌تر شدن TypeScript با آندرس هژلزبرگ
  • ساخت برنامه‌های امن با Azure و Rust
  • یک گفتگوی "بسیار فنی" دیگر با هانسلمن و توب
  • چه چیزهای جدیدی برای توسعه‌دهندگان RPA وجود دارد
  • اشکال‌زدایی پیشرفته Playwright و انعطاف‌پذیری تست
  • شتاب‌بخشی به توسعه چابک بدون باگ و تست خودکار
