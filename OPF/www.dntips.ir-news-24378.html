‫افزونه MSSQL برای VS Code: پروفایلر کوئری، ابزار مهاجرت ADS و موارد بیشتر


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۸ ۰۰:۵۸
آدرس: www.dntips.ir
نسخه جدید v1.40.0 از افزونه MSSQL برای VS Code منتشر شده است که قابلیت‌های جدیدی از جمله ابزار مهاجرت ADS، مدیریت پایگاه داده پایه، وارد کردن فایل‌های متنی، پشتیبان‌گیری و بازیابی پایگاه داده، جستجوی اشیاء پایگاه داده و پروفایلر کوئری را ارائه می‌دهد. این ویژگی‌ها به منظور ساده‌سازی فرآیند مهاجرت از Azure Data Studio، انجام عملیات ضروری پایگاه داده و نظارت بر عملکرد در زمان واقعی، مستقیماً در محیط توسعه Visual Studio Code در دسترس قرار گرفته‌اند. ابزار مهاجرت ADS به کاربران امکان می‌دهد تا اتصالات ذخیره شده، گروه‌های اتصال، تنظیمات و کلیدهای میانبر خود را از Azure Data Studio به VS Code به صورت یکپارچه منتقل کنند. این ابزار شامل افزونه همراه MSSQL Database Management Keymap برای میانبرهای سازگار با ADS مانند F5 برای اجرای کوئری‌ها و Ctrl+L برای برنامه‌های تخمینی است. قابلیت جستجوی اشیاء پایگاه داده امکان یافتن سریع جداول، نماها، رویه‌های ذخیره شده و توابع را در پایگاه‌های داده فراهم می‌کند. کاربران می‌توانند با نام یا پیشوندهای نوع (t:، v:، sp:، f:) جستجو کنند، بر اساس طرح‌واره فیلتر کنند، از یک منوی کشویی پایگاه داده را تغییر دهند و اشیاء را مستقیماً از پنل نتایج اسکریپت کنند. ویژگی‌های مدیریت پایگاه داده پایه (در پیش‌نمایش عمومی) امکان ایجاد، تغییر نام و حذف پایگاه‌های داده را مستقیماً از Object Explorer فراهم می‌کند و از گزینه‌های پیشرفته‌ای مانند collation، recovery model و compatibility level پشتیبانی می‌کند. همچنین، امکان تولید اسکریپت‌های T-SQL برای هر عملیات وجود دارد. پشتیبان‌گیری و بازیابی پایگاه داده (در پیش‌نمایش عمومی) امکان پشتیبان‌گیری از پایگاه‌های داده را به دیسک یا Azure Blob Storage با پشتیبانی از انواع پشتیبان‌گیری کامل، تفاضلی و transaction log فراهم می‌کند. بازیابی از مجموعه‌های پشتیبان‌گیری موجود، فایل‌های .bak یا URLهای Azure Blob Storage با گزینه‌هایی برای قطع اتصالات فعال و مدیریت تاریخچه پشتیبان‌گیری نیز امکان‌پذیر است. پروفایلر کوئری (در پیش‌نمایش عمومی) با استفاده از Extended Events، فعالیت پایگاه داده را در زمان واقعی مستقیماً در VS Code ضبط و نظارت می‌کند. کاربران می‌توانند از الگوهای پروفایلینگ انتخاب کنند، فیلترهای ستونی اعمال کنند، چندین جلسه همزمان را مدیریت کنند، رویدادهای ضبط شده را به CSV صادر کنند و فایل‌های .xel موجود را باز کنند. این قابلیت از SQL Server (در محل و ابری) و Azure SQL Database پشتیبانی می‌کند.
