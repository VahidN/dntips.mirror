‫اقدام سریع‌تر برای راه‌اندازی .NET در GitHub Actions با استفاده از fast-actions/setup-dotnet


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۰۲ ۰۸:۰۸
آدرس: www.dntips.ir
توسعه‌دهنده‌ای به نام _xC0dex به دلیل کند بودن عملکرد اکشن `actions/setup-dotnet` در GitHub Actions، یک جایگزین سریع‌تر به نام `fast-actions/setup-dotnet` ایجاد کرده است. این اکشن با استفاده از دانلودهای موازی، کش کردن و حل هوشمندانه نسخه‌ها، زمان راه‌اندازی .NET را به طور قابل توجهی کاهش می‌دهد. در Scalar، شرکت توسعه‌دهنده، زمان راه‌اندازی .NET از حدود 24 ثانیه به 4 ثانیه کاهش یافته است. این اکشن با استفاده از دستور `uses: fast-actions/setup-dotnet@v1` و تعیین `sdk-version` و `runtime-version` قابل استفاده است. هدف از این اکشن، ارائه یک راه حل سریع‌تر برای راه‌اندازی .NET در GitHub Actions است و قرار نیست جایگزین کامل برای اکشن اصلی باشد. توسعه‌دهنده از بازخورد کاربران قدردانی کرده و `README` را با توضیحات بیشتر به‌روزرسانی کرده است. این اکشن می‌تواند به صرفه‌جویی در زمان و هزینه CI کمک کند، به خصوص زمانی که از runners با محدودیت زمانی استفاده می‌شود. توسعه‌دهندگان می‌توانند با استفاده از این اکشن، فرآیند ساخت و تست پروژه‌های .NET خود را تسریع بخشند.
