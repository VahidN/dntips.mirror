‫فعال‌سازی گزارش‌دهی اشکال‌زدایی در ابزار خط فرمان GitHub (gh)


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۵ ۰۴:۲۸
آدرس: www.dntips.ir
ابزار خط فرمان (CLI) gh یک ابزار قدرتمند برای تعامل با GitHub از خط فرمان است. این ابزار به شما امکان می‌دهد مخازن، مسائل، درخواست‌های ادغام و موارد دیگر را مدیریت کنید بدون اینکه نیاز به ترک ترمینال داشته باشید. با این حال، هنگامی که چیزها طبق انتظار کار نمی‌کنند، درک آنچه در زیربنای آن اتفاق می‌افتد می‌تواند چالش‌برانگیز باشد. گاهی اوقات ممکن است با خطاهایی مبهم مانند پاسخ 403 Forbidden مواجه شوید. این خطا می‌تواند دلایل مختلفی داشته باشد: مجوزهای ناکافی، محدودیت نرخ یا مشکلات احراز هویت. در چنین مواردی، فعال کردن گزارش‌دهی اشکال‌زدایی می‌تواند بینش‌های ارزشمندی در مورد درخواست‌ها و پاسخ‌های HTTP واقعی ارائه دهد. برای فعال کردن گزارش‌دهی اشکال‌زدایی در CLI gh، متغیر محیطی GH_DEBUG را روی 1 یا api تنظیم کنید. این کار اطلاعات دقیق‌تری در مورد درخواست‌های HTTP که انجام می‌شوند، از جمله هدرها، بدنه درخواست و داده‌های پاسخ، نمایش می‌دهد. در لینوکس یا macOS، با استفاده از دستور `export GH_DEBUG=1` و در ویندوز (PowerShell) با استفاده از `$env:GH_DEBUG = "api"` می‌توانید این متغیر را تنظیم کنید. پس از تنظیم این متغیر محیطی، هر دستور gh بعدی شامل اطلاعات اشکال‌زدایی در خروجی خود خواهد بود. به عنوان مثال، با فعال بودن گزارش‌دهی اشکال‌زدایی، دستور `gh release list` اطلاعاتی مانند زمان‌بندی درخواست، URL کامل، هدرهای ارسالی، بدنه درخواست، کد وضعیت HTTP، هدرهای پاسخ و بدنه پاسخ را نمایش می‌دهد. این اطلاعات به شناسایی و رفع مشکلات کمک می‌کند، به خصوص در مواردی که خطاهایی مانند 403 Forbidden رخ می‌دهند.
