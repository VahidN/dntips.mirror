‫تغییر مدل استقرار .NET Framework 3.5 در نسخه‌های جدید ویندوز


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۷ ۰۹:۲۲
آدرس: www.dntips.ir
مایکروسافت مدل استقرار .NET Framework 3.5 را برای نسخه‌های آینده ویندوز تغییر می‌دهد. از Build 27965 ویندوز 11 Insider Preview، .NET Framework 3.5 باید به عنوان یک نصب‌کننده مستقل دریافت شود و دیگر به عنوان یک جزء اختیاری ویندوز گنجانده نمی‌شود. این تغییر برای Build 27965 و نسخه‌های آتی پلتفرم ویندوز اعمال می‌شود و تاثیری بر ویندوز 10 یا نسخه‌های قدیمی‌تر ویندوز 11 تا 25H2 نخواهد داشت. این تغییر با چرخه عمر محصول همسو است، زیرا .NET Framework 3.5 در تاریخ 9 ژانویه 2029 به پایان پشتیبانی خود نزدیک می‌شود و مشتریان را تشویق می‌کند تا مهاجرت به نسخه‌های جدیدتر و پشتیبانی‌شده .NET را برنامه‌ریزی کنند. اگر برنامه شما به .NET Framework 3.5 وابسته است، راهنمایی‌های مفصلی شامل نصب‌کننده‌ها، یادداشت‌های سازگاری و مسیرهای مهاجرت پیشنهادی در Microsoft Learn منتشر شده است. این تغییر به توسعه‌دهندگان توصیه می‌کند که برنامه‌های قدیمی خود را به نسخه‌های جدیدتر .NET منتقل کنند تا از پشتیبانی و امنیت بهتری برخوردار شوند. این اقدام نشان‌دهنده تلاش مداوم مایکروسافت برای مدرن‌سازی پلتفرم .NET و تشویق به استفاده از فناوری‌های جدیدتر است. توسعه‌دهندگان باید برای اطمینان از سازگاری برنامه‌های خود با این تغییرات، برنامه‌ریزی کنند و در صورت نیاز، به نسخه‌های جدیدتر .NET مهاجرت کنند.
