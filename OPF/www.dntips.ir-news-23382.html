‫ساخت اپلیکیشن MCP با C# – نمونه انتخاب‌گر رنگ


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۱ ۰۸:۴۵
آدرس: www.dntips.ir
این مقاله به بررسی نحوه ساخت یک اپلیکیشن MCP با استفاده از C# می‌پردازد. هدف از این آموزش، ایجاد یک رابط کاربری تعاملی برای ابزارهای MCP با استفاده از ASP.NET Core و پروتکل Model Context Protocol (MCP) است. در این نمونه، یک انتخاب‌گر رنگ ساخته می‌شود که به کاربران امکان می‌دهد رنگ‌ها را به صورت بصری انتخاب کنند. در این روش، ابزارهای MCP به جای بازگرداندن متن یا داده‌های ساختاریافته، رابط‌های کاربری تعاملی را برمی‌گردانند که مستقیماً توسط هاست MCP (مانند VS Code) رندر می‌شوند. این امر امکان انتخاب رنگ با استفاده از کلیک، کشیدن و انتخاب بصری را فراهم می‌کند. پیش‌نیازهای این آموزش شامل .NET SDK، VS Code Insiders و آشنایی با ASP.NET minimal APIs است. معماری این نمونه شامل سه بخش اصلی است: سرور MCP + حمل و نقل HTTP، یک ابزار MCP (ColorPicker) و یک منبع MCP که HTML را ارائه می‌دهد (ui://color-picker/app.html). نکته کلیدی در این روش، استفاده از ویژگی [McpMeta] برای تعریف URI منبع UI در ابزار است. ابزار به جای بازگرداندن HTML، داده‌های معمولی و همچنین متادیتا را برمی‌گرداند که به منبع UI پیوند داده می‌شود. برای ایجاد سرور MCP، از یک برنامه ASP.NET Core minimal استفاده شده و MCP فعال شده است. ابزارها و منابع از طریق ویژگی‌ها کشف می‌شوند و همه چیز از طریق HTTP در دسترس قرار می‌گیرد. در نهایت، منبع UI به عنوان یک منبع MCP تعریف می‌شود که با URI مشخص شده مطابقت دارد. این امر به هاست اجازه می‌دهد تا UI را به درستی حل کند. این آموزش یک راهنمای عملی برای توسعه‌دهندگان .NET است که به دنبال استفاده از MCP Apps برای ایجاد ابزارهای تعاملی و رابط‌های کاربری سفارشی هستند. با دنبال کردن این مراحل، می‌توانید یک انتخاب‌گر رنگ ساده را پیاده‌سازی کنید و درک بهتری از نحوه عملکرد MCP Apps به دست آورید.
