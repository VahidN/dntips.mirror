‫به‌‌روزرسانی آمار در Azure SQL Database: زمان و روش


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۰/۱۷ ۰۸:۳۰
آدرس: www.dntips.ir
در Azure SQL Database، آمار دقیق برای عملکرد خوب کوئری‌ها حیاتی است. اگرچه پلتفرم به طور خودکار آمار را در بیشتر سناریوها مدیریت می‌کند، اما در برخی موارد، مداخله دستی نه تنها توصیه می‌شود، بلکه ضروری است. این مقاله توضیح می‌دهد که چه زمانی، چرا و چگونه آمار را در Azure SQL Database به‌روزرسانی کنید، با نمونه‌های عملی و سناریوهای واقعی مشتریان از موارد پشتیبانی تولید. به‌روزرسانی آمار بهینه‌ساز کوئری SQL Server اجازه می‌دهد تا تعداد ردیف‌ها را تخمین بزند، استراتژی‌های اتصال را انتخاب کند، تخصیص حافظه را انجام دهد و تصمیم بگیرد که آیا عملیات را به صورت موازی اجرا کند. آمار قدیمی یا نادرست می‌تواند منجر به کاهش عملکرد کوئری‌ها شود، حتی اگر ایندکس‌گذاری مناسبی وجود داشته باشد. به‌روزرسانی آمار در شرایطی مانند تغییرات بزرگ داده (ETL / عملیات دسته‌ای)، رگرسیون طرح کوئری بدون تغییر طرح و عملیات بازیابی (PITR / Geo-Restore / کپی پایگاه داده) ضروری است. ابزارهایی مانند Query Store می‌توانند به اعتبارسنجی تأثیر به‌روزرسانی آمار کمک کنند.
