عنوان:

‫استفاده از Dev Containers برای توسعه در NET 10.


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۱/۲۴ ۰۸:۰۸
آدرس: www.dntips.ir
مقدمه
در دنیای توسعه نرم‌افزار، نیاز به محیط‌های توسعه منعطف، ایزوله (Isolated) و قابل‌تکرار (Reproducible) بیش از پیش احساس می‌شود. با عرضه نسخه‌های پیش‌نمایش (Preview) NET 10. توسط مایکروسافت، توسعه‌دهندگان به دنبال راه‌هایی هستند تا بدون به‌هم‌ریختن محیط محلی خود، ویژگی‌های جدید این نسخه را آزمایش کنند. در این میان، Dev Containers به‌عنوان راه‌حلی قدرتمند، امکان ایجاد محیط‌های توسعه ایزوله را فراهم می‌کند که نه‌تنها از تداخل با تنظیمات محلی جلوگیری می‌کند، بلکه همکاری تیمی و جابه‌جایی بین محیط‌های مختلف را نیز ساده‌تر می‌سازد. این مقاله به بررسی مفهوم Dev Containers، نحوه تنظیم آن برای NET 10. و نکات کلیدی برای استفاده بهینه از آن می‌پردازد. هدف این است که توسعه‌دهندگان NET. بتوانند با استفاده از این ابزار، تجربه‌ای یکپارچه و کارآمد در آزمایش نسخه‌های جدید داشته باشند.

Dev Containers چیست؟
Dev Containers محیط‌های توسعه‌ای هستند که با استفاده از فناوری کانتینرها (Containers) ایجاد می‌شوند و امکان اجرای پروژه‌ها را در فضایی ایزوله و از پیش‌پیکربندی‌شده فراهم می‌کنند. این محیط‌ها به توسعه‌دهندگان اجازه می‌دهند تا بدون نگرانی از وابستگی‌ها (Dependencies) یا تداخل با سایر ابزارهای نصب‌شده در سیستم محلی، به کدنویسی بپردازند. Dev Containers به‌ویژه در Visual Studio Code و GitHub Codespaces کاربرد گسترده‌ای دارند و از مزایای زیر برخوردارند:
  • ایزولاسیون: محیط کانتینر از سیستم میزبان جدا است و از تداخل با ابزارها یا نسخه‌های دیگر جلوگیری می‌کند.
  • قابلیت تکرار: تنظیمات محیط به‌صورت فایل‌های پیکربندی ذخیره می‌شوند و امکان اشتراک‌گذاری آن‌ها با تیم وجود دارد.
  • قابلیت جابه‌جایی: توسعه‌دهندگان می‌توانند کد خود را در محیط‌های محلی یا ابری با همان تنظیمات اجرا کنند.

این ویژگی‌ها، Dev Containers را به گزینه‌ای ایده‌آل برای آزمایش نسخه‌های جدید NET.، مانند NET 10.، تبدیل کرده‌اند، زیرا توسعه‌دهندگان می‌توانند بدون تغییر در سیستم اصلی خود، ویژگی‌های جدید را کاوش کنند.

Dev Containers مخصوص NET 10.
نسخه NET 10. با ویژگی‌هایی مانند بهبودهای عملکرد، قابلیت‌های جدید در زبان C# و به‌روزرسانی‌هایی در ابزارهای توسعه، نویدبخش تحولاتی در اکوسیستم NET. است. بااین‌حال، نصب نسخه‌های پیش‌نمایش در سیستم محلی ممکن است به دلیل تداخل با نسخه‌های پایدار (LTS) یا نیاز به پیکربندی‌های خاص، چالش‌برانگیز باشد. Dev Containers این مشکل را با ارائه محیطی مستقل حل می‌کند که در آن می‌توان چندین نسخه از NET SDK. را به‌صورت همزمان آزمایش کرد. علاوه بر این، مایکروسافت تصاویر کانتینری (Container Images) متنوعی را برای NET. منتشر می‌کند که در Microsoft Artifact Registry در دسترس هستند و برای سناریوهای مختلف بهینه‌سازی شده‌اند.
برای مثال، تصویر mcr.microsoft.com/dotnet/nightly/sdk:10.0 حاوی آخرین نسخه پیش‌نمایش NET 10. است. برای بررسی جزئیات یک تصویر، می‌توانید از دستور زیر استفاده کنید:
docker inspect mcr.microsoft.com/dotnet/nightly/sdk:10.0
این دستور اطلاعاتی مانند نسخه‌های نصب‌شده و پیکربندی‌های پیش‌فرض را نمایش می‌دهد.

راه‌اندازی Dev Containers برای NET 10.
برای استفاده از Dev Containers در پروژه‌های NET.، مراحل زیر را می‌توان دنبال کرد:
۱. نصب پیش‌نیازها
ابتدا باید ابزارهای زیر نصب شوند:
  • Docker: برای اجرای کانتینرها.
  • Visual Studio Code: به‌عنوان محیط توسعه.
  • افزونه Dev Containers: این افزونه امکان مدیریت کانتینرها را در VS Code فراهم می‌کند.
پس از نصب، پروژه خود را در VS Code باز کنید.

۲. افزودن پیکربندی Dev Container
برای ایجاد فایل‌های پیکربندی، مراحل زیر را انجام دهید:
  1. کلیدهای ترکیبی Ctrl+Shift+P را فشار دهید و گزینه "Dev Containers: Add Development Container Configuration Files…" را انتخاب کنید.
  2. قالب (Template) "C# (.NET)" را انتخاب کنید.
این کار پوشه‌ای به نام .devcontainer در پروژه شما ایجاد می‌کند که شامل فایل devcontainer.json است. این فایل تنظیمات اصلی محیط کانتینر را مشخص می‌کند.

۳. ایجاد Dockerfile سفارشی
برای پشتیبانی از NET 10. Preview، نیاز به یک Dockerfile سفارشی دارید که نسخه‌های موردنظر NET SDK. را نصب کند. نمونه‌ای از یک Dockerfile به‌صورت زیر است:
FROM mcr.microsoft.com/devcontainers/dotnet:1-8.0  # تصویر پایه (LTS)

# نصب NET 9. (STS)
COPY --from=mcr.microsoft.com/dotnet/sdk:9.0 /usr/share/dotnet /usr/share/dotnet

# نصب NET 10. Preview
COPY --from=mcr.microsoft.com/dotnet/nightly/sdk:10.0.100-preview.2 /usr/share/dotnet /usr/share/dotnet
در این فایل:
  • تصویر پایه از نسخه پایدار NET 8. انتخاب شده است.
  • نسخه NET 9. و NET 10. Preview به‌صورت لایه‌ای اضافه شده‌اند.

سپس، فایل devcontainer.json را برای اشاره به Dockerfile به‌روزرسانی کنید:
"build": {
  "dockerfile": "./Dockerfile",
  "context": "."
}

۴. پیکربندی‌های اضافی
برای بهبود تجربه توسعه، می‌توانید تنظیمات زیر را به devcontainer.json اضافه کنید:
  • افزونه‌ها (Extensions): نصب افزونه‌هایی مانند C# DevKit برای پشتیبانی بهتر از NET. در VS Code:
"extensions": ["ms-dotnettools.csdevkit"]
  • ویژگی‌ها (Features): افزودن ابزارهایی مانند Azure CLI:
"features": { "azure-cli": "latest" }
  • دستورات پس از ایجاد (Post-Create Commands): نصب ابزارهای اضافی مانند EF Core:
"postCreateCommand": "dotnet tool install -g dotnet-ef"

۵. ساخت و اجرای Dev Container
برای راه‌اندازی محیط، دستور زیر را اجرا کنید:
  1. کلیدهای Ctrl+Shift+P را فشار دهید و گزینه "Dev Containers: Open Folder in Container" را انتخاب کنید.
  2. پس از ساخت کانتینر، نسخه‌های نصب‌شده NET SDK. را بررسی کنید:
dotnet --list-sdks
این دستور فهرستی از SDKهای موجود در کانتینر را نمایش می‌دهد.

نکته: اگر نیاز به به‌روزرسانی تصویر کانتینر دارید (مثلاً برای دریافت پچ‌های جدید NET.)، از گزینه "Rebuild Container Without Cache" در VS Code استفاده کنید.

نکات کلیدی برای آزمایش محیط‌های ایزوله
  • مدیریت نسخه‌ها: با استفاده از Dev Containers، می‌توانید چندین نسخه NET SDK. را در کنار هم آزمایش کنید بدون اینکه نگران تداخل باشید. این برای مقایسه ویژگی‌های NET 10. با نسخه‌های قبلی بسیار مفید است.
  • اشتراک‌گذاری محیط: فایل‌های .devcontainer را در مخزن پروژه خود ذخیره کنید تا اعضای تیم بتوانند همان محیط را بازسازی کنند.
  • بهینه‌سازی منابع: کانتینرها نسبت به ماشین‌های مجازی سبک‌تر هستند و منابع کمتری مصرف می‌کنند، که باعث افزایش سرعت راه‌اندازی و آزمایش می‌شود.
  • امنیت: محیط‌های ایزوله خطر تغییرات ناخواسته در سیستم محلی را کاهش می‌دهند.

تحقیقات نشان می‌دهد که استفاده از کانتینرها در توسعه نرم‌افزار می‌تواند زمان راه‌اندازی محیط‌های جدید را تا 70٪ کاهش دهد، که این امر به‌ویژه در پروژه‌های تیمی با وابستگی‌های پیچیده اهمیت دارد.

محدودیت‌ها و ملاحظات
هرچند Dev Containers مزایای زیادی دارند، اما محدودیت‌هایی نیز وجود دارد:
  • وابستگی به Docker ممکن است برای سیستم‌های با منابع محدود چالش‌برانگیز باشد.
  • نیاز به دانش اولیه در مورد کانتینرها و Dockerfile برای سفارشی‌سازی پیشرفته‌تر.
  • تصاویر پیش‌نمایش NET. ممکن است ناپایدار باشند، بنابراین باید برای اشکالات احتمالی آماده باشید.

نتیجه‌گیری
Dev Containers راه‌حلی کارآمد و مدرن برای توسعه‌دهندگان NET. ارائه می‌دهد تا بتوانند نسخه‌های جدید مانند NET 10. را در محیطی ایزوله و قابل‌تکرار آزمایش کنند. با تنظیم صحیح Dockerfile، استفاده از افزونه‌های مناسب در VS Code و بهره‌گیری از ویژگی‌های اضافی، می‌توانید تجربه‌ای یکپارچه و بدون دردسر داشته باشید. این رویکرد نه‌تنها از تداخل با محیط محلی جلوگیری می‌کند، بلکه همکاری تیمی را بهبود می‌بخشد و امکان جابه‌جایی بین محیط‌های مختلف را فراهم می‌سازد. توسعه‌دهندگان NET. می‌توانند با اطمینان خاطر، ویژگی‌های جدید NET 10. را کاوش کنند و پروژه‌های خود را با سرعت و دقت بیشتری پیش ببرند.
