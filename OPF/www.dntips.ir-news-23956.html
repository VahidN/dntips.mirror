‫معرفی Windows Sandbox: محیط ایزوله برای تست ایمن برنامه‌ها


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۴ ۰۱:۱۶
آدرس: www.dntips.ir
Windows Sandbox یک محیط ویندوزی یک‌بارمصرف و ایزوله است که از سیستم اصلی شما محافظت می‌کند. این ابزار برای تست نصب‌کننده‌ها، باز کردن پیوست‌های مشکوک یا بررسی ابزارهای جدید بدون ایجاد اختلال در سیستم اصلی بسیار مناسب است. پس از بستن Windows Sandbox، تمام تغییرات آن پاک شده و به حالت اولیه بازمی‌گردد. Windows Sandbox یک جلسه مجازی‌سازی سبک‌وزن از ویندوز است که بر روی سیستم شما اجرا می‌شود و از سیستم عامل اصلی شما جدا است. این محیط شامل یک دسکتاپ تمیز، مجموعه‌ای از فایل‌های سیستم جدید و فضایی برای نصب و اجرای برنامه‌ها است. نکته مهم این است که این محیط پس از بستن، پاک می‌شود و جایگزینی برای ماشین‌های مجازی بلندمدت نیست. برای استفاده از Windows Sandbox، ابتدا باید مطمئن شوید که سیستم شما الزامات لازم را دارد. این الزامات شامل یک نسخه 64 بیتی از ویندوز که از آن پشتیبانی می‌کند، فعال بودن مجازی‌سازی، حداقل 4 گیگابایت رم (8 گیگابایت توصیه می‌شود)، حداقل 1 گیگابایت فضای دیسک آزاد و حداقل 2 هسته پردازنده (4 هسته با Hyper-Threading توصیه می‌شود) است. همچنین، باید بررسی کنید که مجازی‌سازی در BIOS/UEFI فعال باشد. فعال‌سازی Windows Sandbox بسیار ساده است. کافی است در منوی جستجو عبارت "Turn Windows features on or off" را جستجو کرده و گزینه "Windows Sandbox" را فعال کنید. پس از راه‌اندازی، می‌توانید فایل‌ها را به صورت ایمن به داخل Sandbox منتقل کنید، به عنوان مثال با دانلود فایل‌ها از طریق مرورگر داخل Sandbox یا کپی/پیست کردن فایل‌ها از سیستم اصلی. Windows Sandbox با ماشین‌های مجازی (VM) تفاوت دارد. Sandbox سریع‌تر راه‌اندازی می‌شود و به طور خودکار ریست می‌شود، در حالی که VM برای پروژه‌های طولانی‌مدت، گرفتن اسنپ‌شات و استفاده از تصاویر سیستم‌عامل سفارشی مناسب‌تر است. همچنین، Sandbox با "Run as admin" متفاوت است، زیرا Sandbox برنامه‌های پرخطر را از سیستم اصلی شما جدا نگه می‌دارد.
