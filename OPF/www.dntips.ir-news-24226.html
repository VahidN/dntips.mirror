‫پیاده‌سازی سرور MCP با استفاده از .NET SDK و پشتیبانی از VS Code


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۵ ۰۰:۳۸
آدرس: www.dntips.ir
این مقاله نحوه پیاده‌سازی سرور Model Context Protocol (MCP) با استفاده از .NET SDK را به نمایش می‌گذارد. این سرور ابزارهای بررسی سلامت و پیش‌بینی آب‌وهوا را ارائه می‌دهد و با GitHub Copilot در VS Code یکپارچه شده است تا به دستیاران هوش مصنوعی اجازه دهد ابزارهای سفارشی شما را فراخوانی کنند. پروژه یک پیاده‌سازی C# از سرور MCP است که ابزارهای پیش‌بینی آب‌وهوا و بررسی سلامت را در اختیار قرار می‌دهد. این سرور با GitHub Copilot در VS Code ادغام می‌شود و امکان فراخوانی ابزارهای سفارشی توسط دستیاران هوش مصنوعی را فراهم می‌کند. پروژه شامل دو ابزار ping (بررسی سلامت با امکان بازتاب پیام) و get_weather (دریافت پیش‌بینی آب‌وهوا برای شهرها) است. برای شروع، به SDK نسخه 8.0 یا بالاتر از .NET، Visual Studio Code و افزونه GitHub Copilot (با پشتیبانی از MCP) و افزونه C# Dev Kit (توصیه شده) نیاز دارید. ساختار پروژه شامل فایل‌های McpServerTools.cs (تعریف ابزارها)، Program.cs (نقطه ورود سرور) و McpServer.csproj (فایل پروژه) است. همچنین فایل mcp.json برای تنظیمات سرور MCP در پوشه .vscode وجود دارد. برای اجرای سرور، ترمینال را در ریشه پروژه باز کرده و دستور dotnet run --project .\src\McpServer\McpServer\McpServer.csproj را اجرا کنید. پس از راه‌اندازی، می‌توانید با استفاده از Command Palette در VS Code (با فشردن Ctrl + Shift + P یا Cmd + Shift + P) سرور را لیست کرده و به آن متصل شوید. همچنین می‌توانید وضعیت سرور را در پنل خروجی بررسی کنید. برای آزمایش سرور MCP، می‌توانید از ابزارهای ping و get_weather در GitHub Copilot Chat استفاده کنید. به عنوان مثال، با پرسیدن "Ping the server" باید پاسخ "✅ MCP server is alive." را دریافت کنید. این پروژه یک راه حل عملی برای توسعه‌دهندگان .NET است که می‌خواهند ابزارهای سفارشی خود را با استفاده از پروتکل MCP در دسترس مدل‌های هوش مصنوعی قرار دهند.
