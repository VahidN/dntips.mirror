‫فراتر از دستورالعمل‌های سفارشی: سازماندهی آسان اعلان‌های GitHub Copilot


نویسنده: وحید نصیری
تاریخ: ۱۴۰۳/۱۲/۱۸ ۰۷:۳۲
آدرس: www.dntips.ir
این مقاله ویژگی آزمایشی «فایل‌های اعلان قابل استفاده مجدد» (Reusable Prompt Files) در GitHub Copilot را معرفی می‌کند که امکان مدیریت انعطاف‌پذیرتر و کارآمدتر دستورالعمل‌های سفارشی را فراهم می‌کند.

مشکل:
      • دستورالعمل‌های سفارشی (Custom Instructions) به یک فایل Markdown محدود می‌شوند.
      • مدیریت مجموعه‌های متعدد دستورالعمل برای زمینه‌های مختلف دشوار می‌شود.
راه حل:
  • فایل‌های اعلان قابل استفاده مجدد، سازماندهی دستورالعمل‌ها را در چندین فایل Markdown امکان‌پذیر می‌کنند.
    • این فایل‌ها را می‌توان در صورت تقاضا با استفاده از فایل‌های .prompt.md ترکیب کرد.

مزایا:
  • مدیریت ماژولار (Modular Management): دستورالعمل‌ها بر اساس زمینه تقسیم می‌شوند.
    • قابلیت استفاده مجدد بهبود یافته (Improved Reusability): مجموعه‌های دستورالعمل را می‌توان در پروژه‌های مختلف استفاده کرد.
    • تجربه توسعه‌دهنده بهتر (Better Dev Experience): اعمال یکپارچه مجموعه‌های متعدد قوانین از طریق فایل‌های .prompt.md تلفیق شده.

نحوه پیاده‌سازی:
  • chat.promptFiles را در تنظیمات VS Code فعال کنید.
  • فایل‌های Markdown جداگانه‌ای را برای مجموعه‌های دستورالعمل مختلف ایجاد کنید.
  • یک فایل .prompt.md برای وارد کردن و ترکیب این فایل‌ها ایجاد کنید.
  • فایل .prompt.md را در Copilot Chat ضمیمه کنید.

مثال:
  • comment-rule.md برای قراردادهای توضیحات.
  • code-style.md برای خروجی کنسول رنگی.
  • .prompt.md برای وارد کردن هر دو.

توضیحات بیشتر:
  • این ویژگی به توسعه‌دهندگان کمک می‌کند تا دستورالعمل‌های Copilot را به صورت دقیق‌تر و متناسب با نیازهای پروژه خود تنظیم کنند.
  • با استفاده از فایل‌های اعلان قابل استفاده مجدد، می‌توان دستورالعمل‌های پیچیده را به بخش‌های کوچک‌تر و قابل مدیریت‌تر تقسیم کرد.
  • این امر باعث افزایش بهره‌وری و بهبود کیفیت کد تولید شده توسط Copilot می‌شود.
  • به طور مثال میتوان قوانین مربوط به نحوه نام گذاری متغیرها، یا نحوه نوشتن توضیحات کد، یا قوانینی مربوط به نوع خاصی از زبان برنامه نویسی، را در فایل های جداگانه ای قرار داد، و در صورت نیاز هر کدام را فعال کرد.
