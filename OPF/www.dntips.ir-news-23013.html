‫ارسال پارامترها به سرور MCP میزبانی شده در C#


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۰۸ ۰۵:۳۰
آدرس: www.dntips.ir
این مقاله به بررسی روش‌های ارسال پارامترها به سرورهای MCP (Model Context Protocol) میزبانی شده در C# می‌پردازد. MCP امکان یکپارچه‌سازی یکپارچه بین برنامه‌های هوش مصنوعی و منابع داده خارجی را فراهم می‌کند و درک مکانیسم‌های حمل و نقل MCP برای ارسال صحیح پیکربندی ضروری است. دو روش اصلی برای ارتباط سرورهای MCP وجود دارد: STDIO Transport و Hosted MCP Servers. در STDIO Transport، سرور MCP به عنوان یک فرآیند فرزند اجرا می‌شود و از طریق جریان‌های استاندارد ورودی/خروجی ارتباط برقرار می‌کند. در مقابل، Hosted MCP Servers به عنوان سرویس‌های وب مستقل اجرا می‌شوند و از طریق Server-Sent Events (SSE) یا HTTP streaming endpointها به درخواست‌ها پاسخ می‌دهند. برای سرورهای MCP با استفاده از STDIO Transport، ارسال داده‌های پیکربندی نسبتاً ساده است. اما برای سرورهای میزبانی شده، باید از مکانیسم‌های استاندارد API مانند هدرها، پارامترهای URL یا بدنه درخواست استفاده کرد. یک مثال عملی در این مقاله ارائه شده است که در آن پارامترها (نام پروژه و نام تیم) از طریق URL ارسال می‌شوند. کد C# برای استخراج این پارامترها از HttpContext با استفاده از IHttpContextAccessor ارائه شده است. این رویکرد امکان ثبت چندین سرور MCP را با استفاده از پروژه‌ها و نام تیم‌های مختلف فراهم می‌کند. در پایان، مقاله به بهترین شیوه‌ها در ارسال پارامترها به سرورهای MCP اشاره می‌کند، از جمله اهمیت امنیت، رمزگذاری URL و اجتناب از هاردکد کردن اطلاعات حساس.
