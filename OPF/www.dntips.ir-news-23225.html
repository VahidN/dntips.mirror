‫معرفی API پیکربندی کاربر Microsoft Graph (پیش‌نمایش)


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۰۹ ۱۷:۰۶
آدرس: www.dntips.ir
مایکروسافت API پیکربندی کاربر Microsoft Graph را در انتهای بتا به صورت پیش‌نمایش عمومی منتشر کرده است. این API مجموعه‌ای جدید از نقاط پایانی است که به شما امکان می‌دهد اشیاء پیکربندی کاربر را در پوشه‌های Exchange Online ایجاد، خوانده، به‌روزرسانی و حذف کنید. اشیاء پیکربندی کاربر - که به عنوان آیتم‌های مرتبط با پوشه (Folder Associated Items یا FAIs) نیز شناخته می‌شوند - آیتم‌هایی هستند که با یک پوشه خاص مرتبط هستند و هر شیء پیکربندی در یک پوشه باید یک کلید منحصر به فرد داشته باشد. بسیاری از راهکارها به یک روش قابل اعتماد برای ذخیره و بازیابی داده‌های پیکربندی در هر پوشه در کنار محتوای صندوق ورودی نیاز دارند - چه آن حالت برنامه، تنظیمات یا سایر فراداده‌هایی باشد که در سطح پوشه قرار دارند. منبع userConfiguration از چندین سبک payload پشتیبانی می‌کند تا بتوانید داده‌هایی را که به بهترین وجه با سناریوی شما مطابقت دارد ذخیره کنید: binaryData (داده‌های باینری دلخواه)، xmlData (XML سریال شده) و structuredData (جفت‌های کلید-مقدار از انواع داده‌های پشتیبانی شده). این انتشار بتا شامل پشتیبانی کامل CRUD برای اشیاء userConfiguration است: ایجاد، خواندن، به‌روزرسانی و حذف. برای مثال، می‌توانید یک پیکربندی را از کاربر فعال‌العمل (/me) یا یک کاربر خاص (/users/{id})، در سطح یک پوشه ایمیل بخوانید. برای شروع، ساده‌ترین راه، بررسی سطح API و آزمایش درخواست‌ها به صورت تعاملی است. برای کسب اطلاعات بیشتر، مستندات مربوط به نوع منبع userConfiguration و روش‌های Create userConfiguration، Get userConfiguration، Update userConfiguration و Delete userConfiguration را بررسی کنید. همچنین می‌توانید از Graph Explorer برای آزمایش سریع درخواست‌ها استفاده کنید. مایکروسافت از دریافت بازخورد شما استقبال می‌کند و از شما می‌خواهد تا در مورد آنچه که به خوبی کار می‌کند و زمینه‌هایی که می‌توان بهبود بخشید، به ویژه در مورد قابلیت استفاده، شکاف‌ها و سناریوهای دنیای واقعی، بازخورد ارائه دهید.
