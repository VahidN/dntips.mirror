‫ویژگی ویرایش بعدی در GitHub Copilot (پیش‌نمایش)


نویسنده: وحید نصیری
تاریخ: ۱۴۰۳/۱۱/۲۶ ۰۸:۱۷
آدرس: www.dntips.ir
این مطلب سه پیش‌نمایش GitHub Copilot را برای Visual Studio Code معرفی می‌کند: پیشنهادات ویرایش بعدی، حالت عامل برای ویرایش‌های Copilot و Vision. این خلاصه بر پیشنهادات ویرایش بعدی (NES) تمرکز دارد.

پیشنهادات ویرایش بعدی (NES)

  • قابلیت: تکمیل کد Copilot (متن شبح) را به ویرایش کد موجود گسترش می‌دهد. ویرایش‌ها را پیش‌بینی و پیشنهاد می‌کند و گردش‌های کاری اصلاح کد را بهبود می‌بخشد.
    • فعال‌سازی: تنظیم VS Code را فعال کنید: github.copilot.nextEditSuggestions.enabled
    • برای کاربران Copilot Business/Enterprise، مدیران نیز باید "ویژگی‌های پیش‌نمایش ویرایشگر" را انتخاب کنند.
    • VS Code را برای اعمال تغییرات مجدداً بارگذاری کنید.
  • نحوه استفاده: شروع به کدنویسی کنید. پیشنهادات NES به طور خودکار ظاهر می‌شوند. با Tab پیمایش کنید، با Tab بپذیرید.
  • نشانه‌های بصری: یک فلش، نشان دهنده‌ی یک پیشنهاد است. جهت فلش (بالا/پایین) مکان پیشنهاد بعدی را نشان می‌دهد.
  • دامنه پیشنهاد: می‌تواند یک نماد، یک خط یا چندین خط را در بر بگیرد.
    • سناریوهای مثال: اصلاح اشتباه: اشتباهات تایپی و خطاهای منطقی را می‌گیرد (به عنوان مثال، عبارات سه‌گانه معکوس، عملگرهای منطقی نادرست).
    • تغییرات هدف: پیشنهادات را با تغییرات در هدف کدنویسی تطبیق می‌دهد (به عنوان مثال، تغییر نام یک کلاس و به‌روزرسانی کد مرتبط).
    • ادغام کد جدید: به استفاده از متغیرها، روش‌ها یا منطق تازه اضافه شده (حتی در فایل‌ها) کمک می‌کند.
    • بازسازی: به‌روزرسانی‌های مداوم را هنگام تغییر نام یا تغییر الگوهای نامگذاری پیشنهاد می‌کند.
  • بازخورد: نیاز به VS Code Insiders و افزونه پیش‌انتشار GitHub Copilot Chat دارد.
