‫بهینه‌سازی SQL Server Management Studio v21 برای توسعه‌دهندگان


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۳/۰۱ ۰۸:۱۸
آدرس: www.dntips.ir
SQL Server Management Studio (SSMS) ابزاری حیاتی برای توسعه‌دهندگان دات‌نت مایکروسافت و مدیران پایگاه داده است که امکان تعامل با SQL Server را فراهم می‌کند. نسخه‌های جدید SSMS، به ویژه نسخه 21، بر پایه شل (shell) ویژوال استودیو (Visual Studio) ساخته شده‌اند و قابلیت‌های جدید و پیشرفته‌ای را ارائه می‌دهند. پیکربندی صحیح SSMS می‌تواند بهره‌وری شما را به طرز چشمگیری افزایش دهد و تجربه کاربری را بهبود بخشد. این مقاله به بررسی تنظیمات کلیدی در SSMS v21 می‌پردازد که می‌تواند جریان کاری شما را بهینه‌سازی کند و به شما کمک کند تا از تمام پتانسیل این ابزار قدرتمند استفاده کنید.

تنظیمات محیطی (Environment Settings)
پیکربندی محیط کاربری در SSMS اولین گام برای ایجاد یک تجربه کاربری شخصی‌سازی‌شده و کارآمد است. این تنظیمات شامل مواردی مانند تم رنگی، مدیریت اسناد و میانبرهای صفحه کلید می‌شود.

تم رنگی و نمایش اسناد
یکی از اولین تنظیماتی که توجه را جلب می‌کند، گزینه تم رنگی (Color Theme) است. SSMS v21 اکنون حالت تاریک (Dark Mode) را ارائه می‌دهد. با این حال، توجه داشته باشید که ممکن است تمام عناصر محیطی در حالت تاریک به‌طور کامل پیاده‌سازی نشده باشند و تفاوت رنگی بین پنجره‌های مختلف، به ویژه در برنامه‌های اجرایی کوئری (Query Plans)، ممکن است آزاردهنده باشد. تصمیم برای استفاده از حالت تاریک یا حالت پیش‌فرض "Blue" به ترجیح شخصی شما بستگی دارد.

در بخش محیط، اسناد (Environment, Documents)، گزینه "بررسی انتهای خطوط ثابت هنگام بارگذاری" (Check for consistent line endings on load) معمولاً غیرفعال می‌شود. این تنظیم از نمایش هشدارهای مکرر هنگام کار با اسکریپت‌هایی با فرمت‌های متفاوت انتهای خط (line endings) جلوگیری می‌کند، که برای توسعه‌دهندگانی که با کدهای متنوع سروکار دارند، بسیار مفید است.

در بخش محیط، فونت‌ها و رنگ‌ها (Environment, Fonts and Colors)، توصیه می‌شود تنظیمات پیش‌فرض را حفظ کنید. اگرچه فونت‌های کدنویسی متعددی با ظاهر جذاب وجود دارند، اما استفاده از فونت‌های غیرمعمول می‌تواند در هنگام اشتراک‌گذاری اسکرین‌شات‌ها با همکاران یا مشتریان، باعث سردرگمی شود.

میانبرهای صفحه کلید و ویژگی‌های پیش‌نمایش
در بخش محیط، صفحه کلید، میانبرهای کوئری (Environment, Keyboard, Query Shortcuts)، می‌توانید میانبرهای سفارشی برای اسکریپت‌هایی که اغلب اجرا می‌کنید، تعریف کنید. این ویژگی می‌تواند زمان شما را در کارهای تکراری به طرز چشمگیری کاهش دهد. به عنوان مثال، اگر دائماً نیاز به بررسی اطلاعات یک جدول خاص دارید، می‌توانید یک میانبر برای کوئری مربوطه تنظیم کنید.

در بخش محیط، ویژگی‌های پیش‌نمایش (Environment, Preview Features)، گزینه‌ای برای ابطال گواهینامه (Certificate Revocations) وجود دارد که به‌طور پیش‌فرض فعال است. برخی از کاربران معتقدند که غیرفعال کردن این گزینه می‌تواند زمان بارگذاری SSMS را بهبود بخشد، زیرا برنامه دیگر نیازی به بررسی گواهینامه‌های باطل شده ندارد. این تنظیم بیشتر به تجربه شما از سرعت بارگذاری SSMS بستگی دارد.

تنظیمات راه‌اندازی
در بخش گزینه‌های راه‌اندازی (Startup options)، تنظیم SSMS برای باز کردن همزمان "Object Explorer و پنجره کوئری" (Object Explorer and query window) بسیار کارآمد است. به‌طور پیش‌فرض، SSMS تنها Object Explorer را باز می‌کند. با فعال کردن این گزینه، پس از هر بار راه‌اندازی SSMS، یک پنجره کوئری جدید نیز به‌طور خودکار باز می‌شود که برای توسعه‌دهندگانی که بخش عمده‌ای از زمان خود را صرف نوشتن و اجرای کوئری می‌کنند، بسیار مفید است.

پیکربندی تب‌ها و پنجره‌ها (Tab and Window Configuration)
مدیریت کارآمد تب‌ها و پنجره‌ها در SSMS، به ویژه برای توسعه‌دهندگانی که با تعداد زیادی فایل و کوئری به‌طور همزمان کار می‌کنند، اهمیت زیادی دارد.

چیدمان تب‌ها و نام‌گذاری
SSMS v21، با بهره‌گیری از شل ویژوال استودیو، امکانات پیشرفته‌ای برای سازماندهی تب‌ها ارائه می‌دهد. برای بهینه‌سازی فضای کاری، می‌توانید چیدمان تب‌ها را به صورت عمودی (vertical tabs) تنظیم کنید. این کار با رفتن به ابزارها، گزینه‌ها (Tools, Options) و سپس تب‌ها و پنجره‌ها (Tabs and Windows) و تنظیم "چیدمان تب‌ها" (tab layout) به "چپ" (Left) انجام می‌شود.
علاوه بر این، برای خوانایی بیشتر و جلوگیری از شلوغی نام تب‌ها، می‌توانید نام‌گذاری تب‌ها را ساده‌سازی کنید. در بخش ویرایشگر متن، تب ویرایشگر و نوار وضعیت (Text Editor, Editor Tab and Status Bar)، تمام گزینه‌ها را به "False" (نادرست) تنظیم کنید، به جز "شامل نام فایل" (Include file name). این کار باعث می‌شود که تب‌ها تنها نام فایل کوئری را نشان دهند و از نمایش اطلاعات اضافی مانند نام سرور، پایگاه داده یا نام کاربری جلوگیری می‌کند. پس از اعمال این تغییرات، برای مشاهده اثر آن‌ها بر روی تب‌های موجود، نیاز است که SSMS را بسته و مجدداً باز کنید.

پیکربندی کنترل سورس (Source Control Config)
ادغام کنترل سورس با SSMS، به ویژه برای تیم‌های توسعه، ضروری است. SSMS v21 پشتیبانی داخلی از گیت (Git) را ارائه می‌دهد که امکان مدیریت نسخه‌ها و همکاری آسان‌تر را فراهم می‌کند.

تنظیمات جهانی گیت
در بخش کنترل سورس، تنظیمات جهانی گیت (Source Control, Git Global Settings)، می‌توانید نام کاربری و ایمیل حساب گیت‌هاب (Github) خود را پیکربندی کنید. همچنین، باید پوشه‌ای را که مخازن گیت‌هاب خود را در آن نگهداری می‌کنید، مشخص کنید. توصیه می‌شود که این پوشه را خارج از پوشه پیش‌فرض "اسناد" (Documents) سیستم عامل خود قرار دهید. دلیل این کار جلوگیری از کپی شدن تغییرات مداوم به پروفایل‌های رومینگ (roaming profiles) و اشغال فضای پشتیبان‌گیری است. یک پوشه محلی مانند \LocalOnly می‌تواند گزینه‌ای مناسب باشد، زیرا در صورت از کار افتادن رایانه، پشتیبان‌گیری اصلی از طریق خود کنترل سورس انجام می‌شود.
در همین بخش، گزینه‌های "باز کردن پوشه در Solution Explorer هنگام باز کردن یک مخزن گیت" (Open folder in Solution Explorer when opening a Git repository) و "بارگذاری خودکار سلوشن هنگام باز کردن یک مخزن گیت" (Automatically load the solution when opening a Git repository) را فعال کنید. این تنظیمات به‌طور خودکار پوشه و سلوشن (solution) مربوطه را در Solution Explorer بارگذاری می‌کنند، که برای کار بر روی کد منبع در داخل SSMS بسیار کارآمد است.

پیکربندی ویرایشگر متن (Text Editor Config)
تنظیمات ویرایشگر متن، تأثیر مستقیمی بر تجربه کدنویسی و خوانایی کد شما دارد.

تنظیمات عمومی
در بخش ویرایشگر متن، تمام زبان‌ها، عمومی (Text Editor, All Languages, General):
  • شکستن کلمه (Word wrap): فعال کردن این گزینه باعث می‌شود که خطوط طولانی کد به‌طور خودکار شکسته شوند و از اسکرول افقی غیرضروری جلوگیری می‌کند.
  • شماره خطوط (Line numbers): نمایش شماره خطوط در ویرایشگر، برای ارجاع به خطوط خاص کد هنگام بحث با همکاران، بسیار مفید است.

نوارهای اسکرول و فضاهای خالی
در بخش نوارهای اسکرول (Scroll Bars)، گزینه "استفاده از حالت نقشه برای نوار اسکرول عمودی" (Use map mode for vertical scroll bar) یک نقشه کوچک از کد را در نوار اسکرول عمودی نمایش می‌دهد. این ویژگی می‌تواند برای اسکرول سریع در روال‌های ذخیره‌شده (stored procs) طولانی مفید باشد. با نگه داشتن ماوس روی نقشه، یک پنجره پیش‌نمایش از کد در آن نقطه نمایش داده می‌شود.
در بخش ویرایشگر متن، تمام زبان‌ها، تب‌ها (Text Editor, All Languages, Tabs)، اگر ترجیح می‌دهید از فضاهای خالی (spaces) به جای تب‌ها (tabs) برای تورفتگی کد استفاده کنید، گزینه "وارد کردن فضاها" (Insert spaces) را فعال کنید. در حالی که تب‌ها ممکن است برای دسترسی‌پذیری (accessibility) بهتر باشند، انتخاب بین تب و فضا به استانداردها و ترجیحات تیم شما بستگی دارد.

گزینه‌های اجرای کوئری (Query Execution Options)
تنظیمات مربوط به اجرای کوئری می‌تواند بر سرعت کار شما و مدیریت خطرات احتمالی تأثیر بگذارد.

مدیریت کوئری‌ها و تراکنش‌ها
در بخش اجرای کوئری، SQL Server (Query Execution, SQL Server)، توصیه می‌شود دو گزینه آخر را غیرفعال کنید: "درخواست ذخیره کوئری‌های ذخیره نشده" (Prompt to save unsaved queries) و "بررسی تراکنش‌های باز قبل از بستن پنجره‌های کوئری" (Check for open transactions before closing query windows). غیرفعال کردن این گزینه‌ها می‌تواند سرعت بستن پنجره‌های کوئری را افزایش دهد، هرچند ممکن است نیاز به دقت بیشتری در مدیریت تغییرات خود داشته باشید.

یک نکته جالب برای شوخی با همکاران این است که جداکننده دسته (batch separator) را از "GO" به چیزی دیگر مانند "SELECT" تغییر دهید. این کار می‌تواند پیام‌های خطای غیرمنتظره‌ای ایجاد کند و همکاران شما را به چالش بکشد.

نتایج کوئری و نمایش طرح‌ها
در بخش نتایج کوئری، SQL Server، نتایج به گرید (Query Results, SQL Server, Results to Grid)، اندازه داده XML (XML data size) را به "نامحدود" (unlimited) تغییر دهید. این تنظیم برای بازگرداندن طرح‌های اجرایی کوئری (query execution plans) بزرگ بسیار مهم است، که برای تجزیه و تحلیل عملکرد کوئری‌های پیچیده ضروری هستند.
برخی از توسعه‌دهندگان ترجیح می‌دهند گزینه‌های "نمایش نتایج در یک تب جداگانه" (Display results in a separate tab) و "پس از اجرای کوئری، به تب نتایج سوئیچ کن" (Switch to results tab after the query executes) را فعال کنند تا فضای بیشتری برای کوئری و نتایج آن داشته باشند. با این حال، استفاده از میانبر صفحه کلید Control-R برای پنهان کردن یا نمایش پنجره نتایج نیز یک روش کارآمد است.

طراحان و Object Explorer
در بخش طراحان، طراحان جدول و پایگاه داده (Designers, Table and Database Designers)، توصیه می‌شود گزینه "جلوگیری از ذخیره تغییراتی که نیاز به ایجاد مجدد جدول دارند" (Prevent saving changes that require table re-creation) را غیرفعال کنید. این کار به شما اجازه می‌دهد تا تغییرات ساختاری کوچکی را روی جداول اعمال کنید بدون اینکه با محدودیت‌های بازسازی جدول مواجه شوید. این تنظیم به ویژه برای جداول کوچک و در محیط‌های توسعه مفید است.
در بخش SQL Server Object Explorer, دستورات (SQL Server Object Explorer, Commands)، می‌توانید گزینه "احاطه کردن نام شیء با براکت هنگام کشیدن" (Surround object names with brackets when dragged) را غیرفعال کنید. این گزینه به‌طور پیش‌فرض، نام اشیاء را هنگام کشیدن و رها کردن با براکت (square brackets) احاطه می‌کند. اگر نام اشیاء شما حاوی فاصله نیستند، غیرفعال کردن این گزینه می‌تواند مفید باشد، زیرا براکت‌ها ممکن است در برخی موارد (مانند ورودی برای DBCC SHOW_STATISTICS) باعث خطا شوند.

نتیجه‌گیری
پیکربندی دقیق SQL Server Management Studio v21 می‌تواند به طرز چشمگیری تجربه کاری توسعه‌دهندگان دات‌نت را بهبود بخشد. از انتخاب تم رنگی مناسب و سازماندهی کارآمد تب‌ها گرفته تا یکپارچه‌سازی با کنترل سورس و بهینه‌سازی تنظیمات اجرای کوئری، هر یک از این تنظیمات به افزایش بهره‌وری و کاهش اصطکاک در جریان کاری روزمره کمک می‌کنند. با شخصی‌سازی SSMS بر اساس نیازهای خاص خود، می‌توانید ابزاری قدرتمند و بهینه‌سازی‌شده برای تعامل با SQL Server در اختیار داشته باشید.
