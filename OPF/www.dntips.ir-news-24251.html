‫پروتکل زمینه مدل در .NET: ساخت و استفاده از ابزارهای هوش مصنوعی همه‌کاره


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۵ ۰۱:۰۰
آدرس: www.dntips.ir
مایکروسافت اخیراً فریم‌ورک عامل (Agent Framework) را معرفی کرده است که امکان ساخت و مدیریت هوش مصنوعی را در برنامه‌های .NET فراهم می‌کند. این فریم‌ورک شامل مفاهیم و گردش‌کارهای اصلی برای تعامل با دنیای خارج از برنامه است. یکی از چالش‌های اصلی در توسعه هوش مصنوعی، استانداردسازی نحوه تعامل ابزارها با برنامه‌ها است. پیش از این، هر فریم‌ورک هوش مصنوعی روش خاص خود را برای تعریف ابزارها داشت که منجر به ایجاد سیلوها و عدم قابلیت استفاده مجدد ابزارها در فریم‌ورک‌های مختلف می‌شد. پروتکل زمینه مدل (Model Context Protocol - MCP) به عنوان یک استاندارد باز برای حل این مشکل معرفی شده است. MCP یک مشخصات باز است که نحوه دسترسی برنامه‌های هوش مصنوعی به ابزارها و زمینه را استاندارد می‌کند. این پروتکل به عنوان "HTTP برای ابزارهای هوش مصنوعی" شناخته می‌شود و به هر برنامه هوش مصنوعی اجازه می‌دهد تا با استفاده از یک زبان مشترک با ابزارهای مختلف ارتباط برقرار کند. MCP شامل سه جزء اصلی است: سرورها که قابلیت‌های خود را ارائه می‌دهند، کلاینت‌ها که این قابلیت‌ها را مصرف می‌کنند، و یک لایه حمل و نقل انعطاف‌پذیر که ارتباط بین سرور و کلاینت را برقرار می‌کند. Microsoft و Anthropic به طور مشترک یک SDK رسمی برای C# توسعه داده‌اند که به توسعه‌دهندگان .NET امکان می‌دهد به راحتی از MCP استفاده کنند. این SDK شامل کتابخانه‌هایی برای مصرف سرورهای MCP، ساخت سرورهای MCP و یکپارچه‌سازی با ASP.NET Core است. با استفاده از MCP، توسعه‌دهندگان می‌توانند ابزارهای خود را یک بار بسازند و از آن‌ها در هر برنامه MCP-سازگار، از جمله فریم‌ورک عامل، دسکتاپ Claude، VS Code Copilot و برنامه‌های پایتون استفاده کنند. برای شروع، می‌توانید از سرورهای MCP از پیش ساخته شده برای سرویس‌های رایج استفاده کنید. SDK رسمی C# امکان اتصال به سرورهای MCP از طریق روش‌های مختلف حمل و نقل مانند stdio، HTTP/SSE و WebSocket را فراهم می‌کند. این امر به توسعه‌دهندگان .NET اجازه می‌دهد تا به سرعت و به آسانی برنامه‌های هوش مصنوعی خود را با استفاده از MCP توسعه دهند و از قابلیت استفاده مجدد ابزارها بهره‌مند شوند.
