‫راه‌اندازی سریع محیط توسعه با WinGet Configuration


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۱ ۰۹:۲۷
آدرس: www.dntips.ir
WinGet Configuration ابزاری جدید برای راه‌اندازی سریع و خودکار محیط توسعه در ویندوز است. این ابزار به شما امکان می‌دهد پیکربندی محیط توسعه خود را در یک فایل YAML تعریف کرده و با یک دستور، آن را اعمال کنید. این روش جایگزین سریع‌تری نسبت به نصب دستی ابزارها و تنظیمات است. WinGet Configuration با استفاده از ماژول WinGet DSC (Desired State Configuration) کار می‌کند. شما ابتدا باید این ماژول را در PowerShell نصب کنید و سپس فایل پیکربندی YAML خود را با دستور winget configure -f configuration.winget اجرا کنید. WinGet به طور خودکار ابزارها را نصب، تنظیمات را پیکربندی و شما را برای کدنویسی آماده می‌کند. یکی از مزایای WinGet Configuration این است که این پیکربندی‌ها idempotent هستند، به این معنی که می‌توانید آن‌ها را چندین بار اجرا کنید و فقط تغییرات لازم اعمال می‌شوند. برای مثال، اگر Visual Studio Code از قبل نصب شده باشد، WinGet آن را نادیده می‌گیرد و به سراغ مراحل بعدی می‌رود. همچنین می‌توانید از آپشن --accept-configuration-agreements برای رد کردن درخواست‌های تایید خودکار استفاده کنید. WinGet Configuration با winget import/export متفاوت است. winget import/export فقط لیستی از بسته‌های نصب شده را ایجاد می‌کند، در حالی که WinGet Configuration بسیار قدرتمندتر است و می‌تواند تنظیمات ویندوز، فعال‌سازی Developer Mode، نصب workloadsهای Visual Studio، تنظیم متغیرهای محیطی، تعریف وابستگی‌ها، بررسی الزامات سیستم‌عامل و اجرای منابع PowerShell DSC را نیز انجام دهد. به عبارت دیگر، import/export مانند یک لیست خرید است، در حالی که configure مانند یک دستور پخت کامل است. برای سناریوهای ساده نصب برنامه، winget import مناسب است، اما برای راه‌اندازی یک محیط توسعه کامل با تنظیمات و ابزارهای مورد نیاز، استفاده از winget configure توصیه می‌شود.
