‫بازسازی تجربه کامپایل Quake در سال 1997 با Visual C++


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۷ ۲۰:۲۶
آدرس: www.dntips.ir
این مقاله به بررسی مراحل بازسازی تجربه کامپایل نسخه Win32 بازی Quake به شکلی که در سال 1997 انجام می‌شد، می‌پردازد. در ابتدا، توسعه‌دهندگان id Software از سخت‌افزار HP 712-60 با سیستم عامل NeXT و کامپایلر DJGPP بر روی یک سرور DEC Alpha 2100A استفاده می‌کردند، اما پس از مدتی به سمت سخت‌افزار Intergraph با سیستم عامل Windows NT تغییر دادند. هدف از این راهنما، ارائه یک تجربه تاریخی از نحوه ساختن باینری‌های Win32 بازی Quake است. برای دستیابی به بالاترین سطح دقت تاریخی، می‌توان از چهار محیط مختلف استفاده کرد: یک ایستگاه کاری Intergraph RealizM Dual P6-200MHz، یک ماشین دو هسته‌ای Pentium Pro، یک کامپیوتر معمولی اواخر دهه 90 (مانند Quake PC) یا یک ماشین مجازی با استفاده از VirtualBox. نصب Windows NT 4 نسبتاً آسان است و با نمایش تعداد هسته‌های پردازنده و میزان حافظه RAM، یک صفحه شروع مینیمالیستی دارد. برای کامپایل Quake، استفاده از Visual C++ 6 توصیه می‌شود، زیرا نسخه‌های اولیه Quake با Visual C++ 4.X کدنویسی شده بودند. برای این منظور، باید فایل q1source.zip را از آرشیو رسمی Quake دریافت کنید و از باز کردن آن با WinRar نسخه 2.50 استفاده کنید. پس از آن، با استفاده از Visual C++ 6، فضای کاری "WinQuake.dsw" را باز کنید. توجه داشته باشید که برای حفظ یکپارچگی پروژه، نباید کد منبع را از GitHub یا از طریق FTP منتقل کرد، زیرا این کار می‌تواند فایل .dsw را خراب کند و از تجزیه آن توسط Visual C++ 6 جلوگیری کند. در عوض، باید از فایل q1source.zip که در آرشیو رسمی Quake موجود است، استفاده کرد. این روش به توسعه‌دهندگان کمک می‌کند تا تجربه کامپایل بازی Quake را به شکلی مشابه با سال 1997 بازسازی کنند.
