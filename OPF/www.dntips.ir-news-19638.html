‫یک روش مبتکرانه برای بهبود عملکرد پایگاه داده در فراخوانی API ها


نویسنده: فریبرز سیدی
تاریخ: ۱۴۰۳/۰۹/۰۸ ۱۲:۳۲
آدرس: www.dntips.ir
در این ویدیو با ناگت پکیجی (Delta) آشنا می شویم که با اضافه کردن آن می توان عملکرد پایگاه داده را به نحو موثری بهتر کرد. این لایبراری با ترکیب سیستم Sql server change tracking و Http status code 304 تغییرات رو تشخیص میده و از بارگذاری مجدد اطلاعات جلوگیری میکنه. هم با ADO.NET کار میکنه و هم با Entityframework.
بعد از نصب برای فعال کردن آن دو کار لازمه که انجام بشه:
1 - سیستم Sql Server change tracking رو روی پایگاه داده فعال کنید و یا در جداول اطلاعاتی فیلدی از نوع (timestamp) Sqlserver Row Versioning داشته باشید.
2- دستور زیر رو به فایل Program.cs در سمت سرور اضافه کنیم :
app.UseDelta();
و یا اگه API خاصی مد نظرتون هست به شکل زیر کانفیگش کنید:
app.UseDelta(shouldExecute : context => context.Request.Path.ToString().Contains("users"));
در ویدیو به کمک این لایبراری سرعت فراخوانی اطلاعات برای 1 میلیون رکورد در بار اول 980ms هست که در بارگذاری مجدد در صورتیکه تغییری در اطلاعات داده نشده باشه به حدود 12ms کاهش پیدا میکنه. توصیه این است که برای بهترین کارایی از اون برای داده هایی که کمتر اصلاح میشه استفاده بشه.
