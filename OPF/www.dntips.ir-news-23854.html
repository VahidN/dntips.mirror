‫توسعه‌دهنده با استفاده از Ralph Loop و مدل‌های LLM به یک برنامه ویندوزی می‌رسد


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۰۹:۳۳
آدرس: www.dntips.ir
یک توسعه‌دهنده به نام TheJoeFin تجربه‌ای را با استفاده از Ralph Loop و مدل‌های زبانی بزرگ (LLM) برای توسعه یک برنامه ویندوزی به اشتراک گذاشته است. هدف از این پروژه، ایجاد برنامه‌ای بود که استفاده از مدل‌های LLM محلی را آسان کند و از WinUI3 و مدل‌های WinAI استفاده کند. توسعه‌دهنده از Opus 4.5 برای سازماندهی و تعریف مشخصات پروژه استفاده کرده و سپس از Ralph Loop برای اجرای این مشخصات بهره گرفته است. Ralph Loop از Haiku 4.5 برای اجرای وظایف استفاده کرده و پس از چند ساعت و هزاران توکن، برنامه با خطای System.ArgumentException مواجه شد. این خطا نشان‌دهنده مشکلاتی در ایجاد DependencySource در همان Thread بود که DependencyObject در آن قرار دارد. Ralph Loop یک حلقه اسکریپت پایه است که با استفاده از فایل‌های .md برای مدیریت context و جلوگیری از طولانی شدن context در LLM کار می‌کند. این حلقه یک agent را راه‌اندازی کرده و فایل Prompt.md را به آن ارائه می‌دهد که نحوه یافتن و علامت‌گذاری کارها را توضیح می‌دهد. توسعه‌دهنده در نهایت برنامه را به صورت دستی اصلاح کرد تا کار کند، اما UI برنامه هنوز هم دارای مشکلاتی است. توسعه‌دهنده همچنین تلاش کرد تا از Ralph Loop برای توسعه یک برنامه Kanban Files استفاده کند که از فایل‌ها و پوشه‌ها به عنوان آیتم‌ها و Swim Lanes در یک Kanban board استفاده می‌کند. با این حال، Ralph Loop فقط توانست یک shell از برنامه را ایجاد کند و توسعه‌دهنده مجبور شد از طریق چندین reinforcement prompt و استفاده از Opus 4.6 برای نوشتن مشخصات و Sonnet 4.5 برای انجام کارها، مشکلات را برطرف کند.
