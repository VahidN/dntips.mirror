عنوان:

‫اضافه شدن پشتیبانی از Node.js HTTP Servers به Cloudflare Workers


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۶/۳۰ ۰۹:۱۸
آدرس: www.dntips.ir
شرکت کلادفلر (Cloudflare) اخیراً پشتیبانی از APIهای سرور و کلاینت HTTP مربوط به Node.js را به پلتفرم Workers خود اضافه کرده است. این قابلیت مهم به توسعه‌دهندگان اجازه می‌دهد تا اپلیکیشن‌های موجود مبتنی بر Node.js، از جمله فریمورک‌های محبوبی مانند Express.js و Koa.js را به پلتفرم محاسباتی بدون سرور (serverless) کلادفلر منتقل کنند. مزایای اصلی این انتقال شامل حذف کامل «راه‌اندازی سرد» (cold starts)، مقیاس‌پذیری خودکار، و کاهش قابل توجه تأخیر (latency) برای کاربران نهایی است. از نظر فنی، این پیاده‌سازی برای کلاینت HTTP بر پایه API بومی fetch() ساخته شده و برای سرور HTTP، به دلیل عدم امکان ایجاد مستقیم سرورهای TCP در محیط Workers، از یک «سیستم پل ارتباطی» استفاده می‌کند که درخواست‌های ورودی را به سرورهای Node.js ثبت‌شده داخلی متصل می‌کند. با این حال، این رویکرد دارای محدودیت‌هایی نیز هست؛ به عنوان مثال، Agent API غیرفعال است؛ برخی ویژگی‌های پیشرفته HTTP مانند trailers و early hints پشتیبانی نمی‌شوند و گزینه‌های مربوط به TLS به دلیل مدیریت خودکار آن توسط Workers در دسترس نیستند. این ویژگی جدید با استقبال مثبت جامعه توسعه‌دهندگان مواجه شده و آن را راهی برای تسهیل گذار به معماری‌های بدون سرور و تسریع نوآوری می‌دانند.
