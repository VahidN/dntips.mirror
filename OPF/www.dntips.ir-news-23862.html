‫ساخت Agentهای GitHub Copilot با C# و فریم‌ورک Microsoft Agent


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۰۹:۳۹
آدرس: www.dntips.ir
مایکروسافت فریم‌ورک Agent (MAF) را معرفی کرده است که به توسعه‌دهندگان اجازه می‌دهد تا از GitHub Copilot به عنوان یک Agent، نه فقط یک دستیار کدنویسی، استفاده کنند. این فریم‌ورک امکان تعریف اهداف، ابزارها و خودمختاری برای Copilot را فراهم می‌کند و آن را به بخشی از runtime Agent تبدیل می‌کند. در این مقاله، ال برونو (El Bruno) سه نمونه ساده C# را ارائه می‌دهد که نشان می‌دهد چگونه Copilot می‌تواند به عنوان یک Agent مورد استفاده قرار گیرد. این نمونه‌ها شامل ایجاد Agent، تعیین هدف برای آن و استفاده از آن در گردش کار توسعه واقعی است. با استفاده از MAF، Copilot از یک ابزار ساده به یک همکار توسعه‌دهنده تبدیل می‌شود. تغییر مدل ذهنی از Copilot به عنوان یک ابزار به Copilot به عنوان یک Agent، با تعریف اهداف و برنامه‌ریزی برای دستیابی به آن‌ها انجام می‌شود. به جای درخواست مستقیم، وظایف به Agent اختصاص داده می‌شوند. این رویکرد به Copilot اجازه می‌دهد تا به طور مستقل کد را بررسی، تحلیل و پیشنهادهای بهبود را ارائه دهد. این قابلیت‌ها برای توسعه‌دهندگان C# که با GitHub Copilot و سرویس‌های .NET آشنا هستند، بسیار مفید است. MAF کوتاه‌ترین مسیر را از ایده‌های هوش مصنوعی به سیستم‌های واقعی فراهم می‌کند. مستندات مربوط به GitHub Copilot Agent در C# در وب‌سایت مایکروسافت قابل دسترسی است. استفاده از Agentهای Copilot در گردش کار توسعه می‌تواند شامل درک مخازن ناآشنا، توضیح کد قدیمی و ارائه پیشنهادهای بهبود باشد. این قابلیت‌ها به ویژه برای توسعه‌دهندگانی که با پروژه‌های بزرگ و پیچیده کار می‌کنند، ارزشمند هستند.
