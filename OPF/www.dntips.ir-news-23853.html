ШӘЫҢЩ… .NET nanoFramework Ш§Ш№Щ„Ш§Щ… Ъ©ШұШҜЩҮ Ш§ШіШӘ Ъ©ЩҮ ЩҫШҙШӘЫҢШЁШ§ЩҶЫҢ Ш§ШІ Ш§ШҙЪ©Ш§Щ„вҖҢШІШҜШ§ЫҢЫҢ ШҜШұ VS Code ШЁШұШ§ЫҢ Ш§ЫҢЩҶ ЩҒШұЫҢЩ…вҖҢЩҲШұЪ© ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ ШҙШҜЩҮ Ш§ШіШӘ. Ш§ЫҢЩҶ ЩӮШ§ШЁЩ„ЫҢШӘ ШЁЩҮ Ъ©Ш§ШұШЁШұШ§ЩҶ Ш§Щ…Ъ©Ш§ЩҶ Щ…ЫҢвҖҢШҜЩҮШҜ ШӘШ§ Ш§ШІ Щ…ШӯЫҢШ· ШӘЩҲШіШ№ЩҮ Щ…ШӯШЁЩҲШЁ VS Code ЩҲ ЩҮЩ…ЪҶЩҶЫҢЩҶ ШіЫҢШіШӘЩ…вҖҢШ№Ш§Щ…Щ„вҖҢЩҮШ§ЫҢ macOS ЩҲ Linux ШЁШұШ§ЫҢ Ш§ШҙЪ©Ш§Щ„вҖҢШІШҜШ§ЫҢЫҢ ШЁШұЩҶШ§Щ…ЩҮвҖҢЩҮШ§ЫҢ .NET nanoFramework Ш®ЩҲШҜ Ш§ШіШӘЩҒШ§ШҜЩҮ Ъ©ЩҶЩҶШҜ. Ш§ЫҢЩҶ ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ ЩҶЫҢШ§ШІЫҢ ШЁЩҮ ШӘШәЫҢЫҢШұ ШҜШұ Ъ©ШҜ ШҜШіШӘЪҜШ§ЩҮ (firmware) ЩҶШҜШ§ШұШҜ ЩҲ Ш§ШІ ЩҫШұЩҲШӘЪ©Щ„ Ш§ШұШӘШЁШ§Ш·ЫҢ (Wire Protocol) Щ…ЩҲШ¬ЩҲШҜ ШЁЩҮШұЩҮ Щ…ЫҢвҖҢШЁШұШҜ. ЩҫШұЩҲШӘЪ©Щ„ WireШҢ ЫҢЪ© ЩҫШұЩҲШӘЪ©Щ„ ШЁШ§ЫҢЩҶШұЫҢ Ш§ШіШӘ Ъ©ЩҮ ШЁШұШ§ЫҢ ШіЫҢШіШӘЩ…вҖҢЩҮШ§ЫҢ ШӘШ№ШЁЫҢЩҮ ШҙШҜЩҮ ШЁШ§ Щ…ЩҶШ§ШЁШ№ Щ…ШӯШҜЩҲШҜ Ш·ШұШ§ШӯЫҢ ШҙШҜЩҮ Ш§ШіШӘ ЩҲ Ш§Щ…Ъ©Ш§ЩҶШ§ШӘЫҢ Щ…Ш§ЩҶЩҶШҜ ШӘЩҶШёЫҢЩ… ЩҲ ЩҫШ§Ъ© Ъ©ШұШҜЩҶ ЩҶЩӮШ§Ш· ШҙЪ©ШіШӘ (breakpoints)ШҢ ЪҜШ§Щ…вҖҢШЁЩҮвҖҢЪҜШ§Щ… (stepping)ШҢ Щ…ШҜЫҢШұЫҢШӘ ЩҶШ®вҖҢЩҮШ§ (threads) ЩҲ ЩҫШҙШӘЩҮ ЩҒШұШ§Ш®ЩҲШ§ЩҶЫҢ (call stacks)ШҢ ШЁШұШұШіЫҢ ЩҲ ШӘШәЫҢЫҢШұ Щ…ШӘШәЫҢШұЩҮШ§ШҢ ЩҲ Щ…ШҜЫҢШұЫҢШӘ Ш§ШіШӘШ«ЩҶШ§ (exception handling) ШұШ§ ЩҒШұШ§ЩҮЩ… Щ…ЫҢвҖҢЪ©ЩҶШҜ. ШӘЫҢЩ… ШӘЩҲШіШ№ЩҮШҢ ШЁШұШ§ЫҢ ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ Ш§ЫҢЩҶ ЩӮШ§ШЁЩ„ЫҢШӘШҢ Ш§ШІ Ъ©ШҜ Щ…ЩҶШЁШ№ Ш§ЩҒШІЩҲЩҶЩҮ Visual Studio (nf-Visual-Studio-extension) ШЁЩҮ Ш№ЩҶЩҲШ§ЩҶ ЫҢЪ© Щ…ЩҶШЁШ№ ШӯЫҢШ§ШӘЫҢ Ш§ШіШӘЩҒШ§ШҜЩҮ Ъ©ШұШҜЩҮ Ш§ШіШӘ. Ш§ЫҢЩҶ Ш§ЩҒШІЩҲЩҶЩҮ ШіШ§Щ„вҖҢЩҮШ§ШіШӘ Ъ©ЩҮ Ш§ШІ Ш§ШҙЪ©Ш§Щ„вҖҢШІШҜШ§ЫҢЫҢ ШҜШұ Visual Studio ЩҫШҙШӘЫҢШЁШ§ЩҶЫҢ Щ…ЫҢвҖҢЪ©ЩҶШҜ ЩҲ ШҜШұЪ© ЩҶШӯЩҲЩҮ Ш№Щ…Щ„Ъ©ШұШҜ ШўЩҶ ШЁЩҮ ШӘЫҢЩ… Ъ©Щ…Ъ© ШҙШ§ЫҢШ§ЩҶЫҢ Ъ©ШұШҜЩҮ Ш§ШіШӘ. ШЁШ§ ШЁШұШұШіЫҢ Ш§ЫҢЩҶ Ъ©ШҜШҢ ШӘЫҢЩ… ШӘЩҲШіШ№ЩҮ ШӘЩҲШ§ЩҶШіШӘ ЩҶШӯЩҲЩҮ Ш§ШіШӘЩӮШұШ§Шұ Ш§ШіЩ…ШЁЩ„ЫҢвҖҢЩҮШ§ ШҜШұ ШҜШіШӘЪҜШ§ЩҮШҢ ШӘШұШ¬Щ…ЩҮ ЩҶЩӮШ§Ш· ШҙЪ©ШіШӘ ШЁЩҮ Щ…Ъ©Ш§ЩҶвҖҢЩҮШ§ЫҢ ЩӮШ§ШЁЩ„ ЩҒЩҮЩ… ШЁШұШ§ЫҢ ШҜШіШӘЪҜШ§ЩҮШҢ ЩҲ ЩҫЫҢЩ…Ш§ЫҢШҙ ШіЫҢШіШӘЩ… Щ…ЩӮШ§ШҜЫҢШұ runtime ШҜШіШӘЪҜШ§ЩҮ ШұШ§ ШҜШұЪ© Ъ©ЩҶШҜ. Ш§ЫҢЩҶ ЩӮШ§ШЁЩ„ЫҢШӘ ШЁШ§ Ш§ШіШӘЩҒШ§ШҜЩҮ Ш§ШІ ЫҢЪ© Щ„Ш§ЫҢЩҮ ШӘШұШ¬Щ…ЩҮ ШЁЫҢЩҶ ЩҫШұЩҲШӘЪ©Щ„ Wire ЩҲ ЩҫШұЩҲШӘЪ©Щ„ Debug Adapter Protocol (DAP) Ъ©ЩҮ ШӘЩҲШіШ· VS Code Ш§ШіШӘЩҒШ§ШҜЩҮ Щ…ЫҢвҖҢШҙЩҲШҜШҢ Ш§Щ…Ъ©Ш§ЩҶвҖҢЩҫШ°ЫҢШұ ШҙШҜЩҮ Ш§ШіШӘ. Ъ©ШӘШ§ШЁШ®Ш§ЩҶЩҮ nf-debugger ЩҶЩӮШҙ Ъ©Щ„ЫҢШҜЫҢ ШҜШұ Ш§ЫҢЩҶ ЩҒШұШўЫҢЩҶШҜ Ш§ЫҢЩҒШ§ Щ…ЫҢвҖҢЪ©ЩҶШҜ ЩҲ ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ Ъ©Ш§Щ…Щ„ЫҢ Ш§ШІ ЩҫШұЩҲШӘЪ©Щ„ Wire ШұШ§ Ш§ШұШ§ШҰЩҮ Щ…ЫҢвҖҢШҜЩҮШҜ. Ш§ЫҢЩҶ Ъ©ШӘШ§ШЁШ®Ш§ЩҶЩҮ ШЁШ§ ЫҢЪ© ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ ШіШ·Шӯ ЩҫШ§ЫҢЫҢЩҶ C/C++ ШҜШұ ШҜШіШӘЪҜШ§ЩҮ Ш§ШұШӘШЁШ§Ш· ШЁШұЩӮШұШ§Шұ Щ…ЫҢвҖҢЪ©ЩҶШҜ. Ш§ЫҢЩҶ ШЁЩҮвҖҢШұЩҲШІШұШіШ§ЩҶЫҢ ЫҢЪ© ЪҜШ§Щ… Щ…ЩҮЩ… ШҜШұ Ш¬ЩҮШӘ ШЁЩҮШЁЩҲШҜ ШӘШ¬ШұШЁЩҮ ШӘЩҲШіШ№ЩҮвҖҢШҜЩҮЩҶШҜЪҜШ§ЩҶ .NET nanoFramework Ш§ШіШӘ ЩҲ Ш§Щ…Ъ©Ш§ЩҶШ§ШӘ ШЁЫҢШҙШӘШұЫҢ ШұШ§ ШЁШұШ§ЫҢ ШӘЩҲШіШ№ЩҮ ШЁШұЩҶШ§Щ…ЩҮвҖҢЩҮШ§ЫҢ ШӘШ№ШЁЫҢЩҮ ШҙШҜЩҮ ШЁШ§ Ш§ШіШӘЩҒШ§ШҜЩҮ Ш§ШІ C# ЩҒШұШ§ЩҮЩ… Щ…ЫҢвҖҢЪ©ЩҶШҜ.

    Ш§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§NET nanoFramework. ШЁШұШ§ЫҢ ШӘЩҲШіШ№ЩҮвҖҢЫҢ ШіЫҢШіШӘЩ…вҖҢЩҮШ§ЫҢ embeddedШ§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§SharpDbg: Ш§ШҙЪ©Ш§Щ„вҖҢШІШҜШ§ (Debugger) Ш¬ШҜЫҢШҜ ЩҲ ЪҶЩҶШҜШіЪ©ЩҲЫҢЫҢ .NET ШЁШ§ ШІШЁШ§ЩҶ C#!Ш§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§ЩҫЫҢШ§ШҜЩҮвҖҢШіШ§ШІЫҢ ШіШұЩҲШұ MCP ШЁШ§ Ш§ШіШӘЩҒШ§ШҜЩҮ Ш§ШІ .NET SDK ЩҲ ЩҫШҙШӘЫҢШЁШ§ЩҶЫҢ Ш§ШІ VS CodeШ§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§ШіШұЩҲШұЩҮШ§ЫҢ MCP Ш§ШІ ШұШ§ЩҮ ШҜЩҲШұ ШЁШ§ Ш§ШіШӘЩҒШ§ШҜЩҮ Ш§ШІ .NET SDK вҖ“ ЫҢЪ©ЩҫШ§ШұЪҶЩҮвҖҢШіШ§ШІЫҢ ШЁШ§ ШҜШ§ШҜЩҮвҖҢЩҮШ§ ЩҲ APIЩҮШ§ЫҢ ШіЩҒШ§ШұШҙЫҢШ§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§Щ…Ш¬Щ…ЩҲШ№ЩҮ Щ…ЩӮШ§Щ„Ш§ШӘ Ш¬ШҜЫҢШҜ: ШіШ§Ш®ШӘ ЫҢЪ© ШҜШіШӘЪҜШ§ЩҮ ШӘШ№ШЁЫҢЩҮ ШҙШҜЩҮ ШҜШұ ШіШ·Шӯ ШөЩҶШ№ШӘЫҢ ШЁШ§ .NET ЩҲ Uno PlatformШ§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§ЩҫЫҢШҙвҖҢЩҶЩ…Ш§ЫҢШҙ ЩҫШҙШӘЫҢШЁШ§ЩҶЫҢ Ъ©Ш§ШӘЩ„ЫҢЩҶ ШЁШұШ§ЫҢ ЩҲЫҢЪҳЩҲШ§Щ„ Ш§ШіШӘЩҲШҜЫҢЩҲ Ъ©ШҜШ§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§Щ…Ш№ШұЩҒЫҢ Щ…Ш®ШІЩҶ В«MCP for BeginnersВ»Ш§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§ШЁШұШұШіЫҢ ШөШ§ШҜЩӮШ§ЩҶЩҮ Ш§ШІ IDEЩҮШ§ЫҢ Щ…ШҜШұЩҶ ЩҲ ШӘЩҲШіШ№ЩҮ Щ…ШЁШӘЩҶЫҢ ШЁШұ ЩҮЩҲШҙ Щ…ШөЩҶЩҲШ№ЫҢ ШҜШұ .NETШ§ШҙШӘШұШ§Ъ©вҖҢЩҮШ§ЩҫШұЩҲЪҳЩҮ MsQuic Щ…Ш§Ъ©ШұЩҲШіШ§ЩҒШӘ ШіЩҲШұШі ШЁШ§ШІ ШҙШҜЩ…Ш·Ш§Щ„ШЁЪҶШұШ§ MCPШҹ (ШЁШ®Шҙ ШҜЩҲЩ…- LSP)
