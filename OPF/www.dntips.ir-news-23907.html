‫استفاده از Microsoft Agent Framework با Ollama در .NET/C#


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۱۲:۴۸
آدرس: www.dntips.ir
این مقاله به بررسی نحوه استفاده از Microsoft Agent Framework با Foundry Local می‌پردازد. به دلیل عدم پشتیبانی Foundry Local از فراخوانی ابزارها به صورت native، پیاده‌سازی دستی حلقه فراخوانی ابزارها ضروری است. در این روش، مدل زبانی بزرگ (LLM) ابتدا لیستی از ابزارهای موجود دریافت می‌کند و تصمیم می‌گیرد که آیا فراخوانی ابزاری لازم است یا خیر. سپس پاسخ مدل تجزیه و تحلیل شده، ابزار انتخاب شده اجرا می‌شود و نتایج به LLM بازخورد داده می‌شود تا تولید نهایی انجام شود. این رویکرد با الهام از مقاله "Building Multi-Agent Workflows in .NET with AgentFactory and Handoff" توسط El Bruno انجام شده است که استفاده از Microsoft Agent Framework با Ollama را از طریق provider native آن معرفی می‌کند. هدف این مقاله، گسترش نمونه قبلی با افزودن Ollama به عنوان یک provider اضافی است تا امکان جابجایی بین Foundry Local و Ollama بر اساس نیازهای مختلف فراهم شود. برای شروع، باید Ollama را بر روی سیستم ویندوز 11 نصب کنید و مدل mistral را دانلود کنید. سپس، تنظیمات پروژه Aspire شامل تعریف active provider (Ollama)، endpoint Ollama و مدل مورد استفاده است. در کد، AGENT_PROVIDER، OLLAMA_ENDPOINT، OLLAMA_MODEL و MCP_TOOLS به عنوان environment variables تنظیم می‌شوند تا پیکربندی به صورت پویا انجام شود. این روش به توسعه‌دهندگان .NET اجازه می‌دهد تا به راحتی از قابلیت‌های LLM در برنامه‌های خود استفاده کنند و با انتخاب provider مناسب، انعطاف‌پذیری بیشتری در طراحی و پیاده‌سازی داشته باشند. استفاده از Ollama به عنوان یک provider، امکان استفاده از مدل‌های زبانی مختلف را بدون نیاز به تغییرات اساسی در کد فراهم می‌کند. در نهایت، این مقاله یک راهنمای عملی برای ادغام Ollama با Microsoft Agent Framework در پروژه‌های .NET ارائه می‌دهد و به توسعه‌دهندگان کمک می‌کند تا از مزایای هر دو ابزار به طور همزمان بهره‌مند شوند.
