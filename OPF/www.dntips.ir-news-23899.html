‫بهبود عملکرد کوئری‌های کند در EF Core


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۱۲:۴۰
آدرس: www.dntips.ir
در این مقاله به بررسی دلایل کندی کوئری‌های EF Core و ارائه راهکارهایی برای رفع این مشکل می‌پردازیم. استفاده از EF Core به دلیل انعطاف‌پذیری و قدرت آن بسیار رایج است، اما نوشتن کوئری‌های ناکارآمد به صورت ناخواسته نیز آسان است که می‌تواند منجر به کندی برنامه ASP.NET Core یا کاهش عملکرد سرور SQL Server شود. یکی از دلایل اصلی کندی کوئری‌ها، استفاده از نوع داده nvarchar(max) برای ستون‌های رشته‌ای است. به طور پیش‌فرض، EF Core تمام ویژگی‌های رشته‌ای را به عنوان طول نامحدود در نظر می‌گیرد و در نتیجه، ستون‌های nvarchar(max) در پایگاه داده ایجاد می‌شوند. این نوع ستون‌ها نمی‌توانند به طور موثر ایندکس شوند و در نتیجه، کوئری‌ها و مرتب‌سازی‌ها کندتر می‌شوند. برای رفع این مشکل، باید طول حداکثر برای ستون‌های رشته‌ای مشخص شود، به عنوان مثال با استفاده از متد HasMaxLength در فایل‌های پیکربندی DbContext. دلیل دیگر کندی کوئری‌ها، بازیابی کل موجودیت‌ها (Entities) است. اگر فقط به چند فیلد از یک موجودیت نیاز دارید، بازیابی کل موجودیت می‌تواند غیرضروری و ناکارآمد باشد. برای رفع این مشکل، می‌توان یک کلاس DTO (Data Transfer Object) جداگانه ایجاد کرد و فقط فیلدهای مورد نیاز را انتخاب کرد. این کار باعث کاهش حجم داده‌های منتقل شده و بهبود عملکرد کوئری می‌شود. در نهایت، با اعمال این تغییرات و بهینه‌سازی کوئری‌ها، می‌توان عملکرد برنامه ASP.NET Core را به طور قابل توجهی بهبود بخشید و از بار اضافی بر روی سرور SQL Server جلوگیری کرد. استفاده از این روش‌ها به توسعه‌دهندگان ‎.NET کمک می‌کند تا برنامه‌های کارآمدتر و مقیاس‌پذیرتری ایجاد کنند.
