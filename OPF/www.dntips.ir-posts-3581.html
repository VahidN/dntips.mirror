‫نصب و پیکربندی Postfix برای ارسال ایمیل - قسمت ۴ - عیب‌یابی اولیه با PTR Record


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۴/۱۸ ۰۸:۱۰
آدرس: www.dntips.ir
در قسمت قبل، اولین ایمیل آزمایشی خود را ارسال کردیم. ممکن است ایمیل با موفقیت به اینباکس رسیده باشد، یا شاید مستقیم به پوشه اسپم رفته باشد. یکی از مقصران اصلی برای اسپم شدن ایمیل‌ها، عدم تنظیم صحیح رکوردی به نام PTR است. در این فصل، با این رکورد مهم آشنا می‌شویم، نحوه بررسی آن را یاد می‌گیریم و روش صحیح تنظیم آن را فرا خواهیم گرفت.

بخش ۴.۱: نصب dnsutils

برای تمام بررسی‌های مربوط به DNS، ما از ابزار قدرتمند dig استفاده می‌کنیم. همانطور که در فصل ۲ اشاره شد، اگر این ابزار هنوز روی سرور شما نصب نیست، ابتدا باید بسته dnsutils را نصب کنید. اگر قبلاً این کار را انجام داده‌اید، می‌توانید از این مرحله عبور کنید.
برای نصب، دستور زیر را اجرا کنید:
sudo apt install dnsutils -y

بخش ۴.۲: PTR Record چیست؟ (نگاهی به DNS معکوس)

تا اینجا ما با جستجوی "DNS مستقیم" (Forward DNS) کار کرده‌ایم. یعنی یک نام دامنه (مانند mail.yourdomain.com) را به یک آدرس IP نگاشت دادیم.
رکورد PTR (Pointer) دقیقاً برعکس این کار را انجام می‌دهد. این رکورد یک آدرس IP را به یک نام دامنه نگاشت می‌دهد. این فرآیند به "جستجوی DNS معکوس" (Reverse DNS Lookup) معروف است.

چرا اینقدر مهم است؟
وقتی سرور شما با آدرس IP 203.0.113.50 به سرور Gmail متصل می‌شود و می‌گوید: "سلام، من mail.yourdomain.com هستم"، سرور Gmail برای راستی‌آزمایی این ادعا، یک جستجوی معکوس روی IP 203.0.113.50 انجام می‌دهد. او از سیستم DNS می‌پرسد: "صاحب این IP کیست؟"
  • حالت ایده‌آل: سیستم DNS پاسخ می‌دهد: "صاحب این IP، mail.yourdomain.com است." در این حالت، ادعای سرور شما تأیید می‌شود و یک امتیاز مثبت بزرگ برای اعتبار ایمیل شما ثبت می‌شود.
  • حالت بد (و بسیار رایج): سیستم DNS یک نام عمومی و بی‌ربط مانند static.50.113.0.203.clients.your-hosting-provider.com را برمی‌گرداند یا اصلاً پاسخی نمی‌دهد. در این حالت، سرور Gmail به ادعای سرور شما مشکوک شده و به احتمال زیاد ایمیل را به عنوان اسپم شناسایی می‌کند، زیرا هویت فرستنده با IP آن همخوانی ندارد.

عدم تنظیم صحیح رکورد PTR یکی از دلایل قطعی برای رد شدن یا اسپم شدن ایمیل‌ها توسط سرورهای بزرگ است.

بخش ۴.۳: بررسی PTR Record با دستور dig

حالا بیایید بررسی کنیم که آیا برای IP سرور ما رکورد PTR به درستی تنظیم شده است یا خیر. برای این کار، از دستور dig با آپشن -x (که مخفف reverse lookup است) استفاده می‌کنیم.
دستور زیر را در ترمینال اجرا کنید و YOUR_SERVER_IP را با آدرس IP عمومی سرور خود جایگزین نمایید:
dig -x YOUR_SERVER_IP +short
تفسیر نتایج:
۱. نتیجه صحیح و ایده‌آل:
خروجی دستور باید دقیقاً همان نام میزبانی (FQDN) باشد که در فصل اول تنظیم کردید.
mail.yourdomain.com.

۲. نتیجه اشتباه یا عدم تنظیم:
اگر خروجی چیزی شبیه به موارد زیر بود، یعنی رکورد PTR شما نیاز به اصلاح دارد:
# خروجی یک نام عمومی از طرف شرکت میزبان
1-2-3-4.static.provider.net.

# یا خروجی خالی که به معنی عدم وجود رکورد PTR است

بخش ۴.۴: چگونه PTR Record را تنظیم کنیم؟

نکته بسیار مهمی که باید بدانید این است که شما نمی‌توانید رکورد PTR را از طریق پنل DNS دامنه خود تنظیم کنید. رکوردهای A و MX توسط صاحب دامنه مدیریت می‌شوند، اما رکورد PTR توسط صاحب بلوک آدرس IP مدیریت می‌شود که در ۹۹٪ موارد، همان شرکت ارائه‌دهنده سرور (Hosting Provider) شماست.
بنابراین، برای تنظیم این رکورد باید با شرکت میزبان خود ارتباط برقرار کنید. مراحل کار به شرح زیر است:

مرحله ۱: بررسی پنل کاربری شرکت میزبان
بسیاری از ارائه‌دهندگان سرور مدرن (مانند DigitalOcean, Vultr, Linode, Hetzner) به شما اجازه می‌دهند که رکورد PTR (که گاهی اوقات Reverse DNS یا rDNS نامیده می‌شود) را مستقیماً از طریق پنل کاربری خود برای IP سرورتان تنظیم کنید. به دنبال گزینه‌هایی با این نام‌ها در بخش مدیریت شبکه یا مدیریت سرور خود بگردید. اگر این گزینه را پیدا کردید، به سادگی مقدار آن را برابر با نام میزبان کامل سرور خود قرار دهید: mail.yourdomain.com.

مرحله ۲: ارسال تیکت پشتیبانی (در صورت عدم وجود پنل)
اگر گزینه‌ای برای تنظیم مستقیم پیدا نکردید، باید یک درخواست پشتیبانی (Support Ticket) برای تیم فنی شرکت میزبان ارسال کنید. در ادامه یک نمونه متن درخواست به زبان انگلیسی آمده است که می‌توانید از آن استفاده کنید:
Subject: Please set PTR Record for my server IP

Hello Support Team,

I am configuring a mail server and I need to set the PTR record (Reverse DNS) for my server's IP address.

Could you please point the following IP address to my mail server's hostname?

- **IP Address:** YOUR_SERVER_IP
- **Hostname to point to:** mail.yourdomain.com

This is required to ensure email deliverability and prevent our emails from being marked as spam.

Thank you for your assistance.

Best regards,
[Your Name]
اطمینان حاصل کنید که YOUR_SERVER_IP و mail.yourdomain.com را با مقادیر صحیح جایگزین کرده‌اید.
پس از اینکه شرکت میزبان تغییرات را اعمال کرد، ممکن است چند ساعت طول بکشد تا به‌روزرسانی در سراسر اینترنت منتشر شود. شما می‌توانید مجدداً با دستور dig -x YOUR_SERVER_IP وضعیت را بررسی کنید تا از صحت آن مطمئن شوید.
با تنظیم صحیح این رکورد، شما یک گام بسیار بزرگ در جهت افزایش اعتبار سرور ایمیل خود برداشته‌اید و آماده‌اید تا به سراغ لایه‌های امنیتی بعدی یعنی SPF و DKIM بروید.
