‫اعلام SDK جدید 3.x برای Application Insights در .NET


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۰۹:۳۶
آدرس: www.dntips.ir
مایکروسافت متعهد به استفاده از OpenTelemetry به عنوان مبنای اصلی قابلیت مشاهده (Observability) در Azure است. در همین راستا، نسخه جدید SDK 3.x برای Application Insights در ‎.NET منتشر شده است که مهاجرت به OpenTelemetry را برای توسعه‌دهندگان آسان‌تر می‌کند. این نسخه جدید به توسعه‌دهندگان اجازه می‌دهد تا با ارتقاء نسخه SDK خود، به سادگی به OpenTelemetry مهاجرت کنند. در نسخه‌های قبلی، مهاجرت از SDK کلاسیک Application Insights به توزیع OpenTelemetry نیازمند نصب مجدد و به‌روزرسانی کد بود. اما با این نسخه جدید، بیشتر مشتریان می‌توانند با ارتقاء SDK خود، از OpenTelemetry بهره‌مند شوند. SDK جدید به طور خودکار فراخوانی‌های APIهای کلاسیک Application Insights مانند Track* را از طریق یک لایه نگاشت جدید هدایت می‌کند که سیگنال‌های OpenTelemetry را در زیرساخت تولید می‌کند. این امر دسترسی به APIهای OpenTelemetry، کتابخانه‌های ابزارسازی جامعه و پشتیبانی از چند صادرکننده (Multi-exporter) را فراهم می‌کند. در حالی که OpenTelemetry استفاده از OpenTelemetry Collector را تشویق می‌کند، مایکروسافت به حفظ سادگی Application Insights متعهد است. توزیع Azure Monitor OpenTelemetry Distro تنها بسته‌ای است که برای شروع لازم است و با یک رشته اتصال (Connection String) قابل پیکربندی است. این امر روح "کار می‌کند" (just works) Application Insights را حفظ می‌کند، در حالی که با استانداردهای OpenTelemetry همگام است. برای دریافت بازخورد یا ارتباط مستقیم با تیم محصول، می‌توانید به آدرس otel@microsoft.com ایمیل ارسال کنید. در صورت بروز مشکل در هنگام ارتقاء، لطفاً یک درخواست پشتیبانی (support ticket) باز کنید.
