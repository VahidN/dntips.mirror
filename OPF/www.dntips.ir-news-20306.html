‫معرفی Incrementalist؛ ابزاری برای کاهش زمان Build پروژه‌های بزرگ، از یک ساعت به 15 دقیقه


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۱/۳۰ ۰۸:۴۶
آدرس: www.dntips.ir
مقدمه
در دنیای توسعه نرم‌افزار، به‌ویژه در پروژه‌های بزرگ مبتنی بر NET.، بهینه‌سازی فرآیند ساخت (Build) و تست (Test) از اهمیت بسزایی برخوردار است. با افزایش پیچیدگی پروژه‌ها، مانند راه‌حل‌های چندپروژه‌ای (Multi-Project Solutions)، مخازن تک‌ریشه‌ای (Monorepos) یا معماری‌های میکروسرویس (Microservice Architectures)، زمان اجرای خطوط لوله CI/CD (پیوسته‌سازی ادغام و استقرار) به چالش بزرگی تبدیل می‌شود. ابزارهایی مانند Incrementalist با بهره‌گیری از تحلیل تغییرات مبتنی بر Git، راهکاری برای کاهش زمان ساخت و تست ارائه می‌دهند.

مفهوم Incrementalist و اهمیت آن
Incrementalist ابزاری مبتنی بر NET. است که با استفاده از کتابخانه‌های LibGit2Sharp و Roslyn، تغییرات کد در مخزن Git را تحلیل کرده و تنها پروژه‌های تحت تأثیر تغییرات را شناسایی، ساخته و تست می‌کند. این ابزار برای سناریوهای پیچیده‌ای مانند راه‌حل‌های بزرگ (Large Solutions)، مخازن تک‌ریشه‌ای و پروژه‌هایی با وابستگی‌های پیچیده طراحی شده است. برخلاف روش‌های سنتی که کل راه‌حل را در هر ساخت بازسازی می‌کنند، Incrementalist با تمرکز بر تغییرات افزایشی (Incremental Changes)، زمان اجرای خطوط لوله CI/CD را به‌طور قابل‌توجهی کاهش می‌دهد.

مزایای اصلی Incrementalist عبارت‌اند از:
  • کاهش زمان ساخت و تست: با محدود کردن فرآیند به پروژه‌های تغییر یافته، زمان اجرای خطوط لوله کوتاه‌تر می‌شود.
  • پشتیبانی از سناریوهای پیچیده: مناسب برای مخازن تک‌ریشه‌ای و معماری‌های میکروسرویس.
  • انعطاف‌پذیری در اجرا: امکان اجرای دستورات به‌صورت موازی یا ذخیره لیست پروژه‌های تحت تأثیر برای استفاده‌های بعدی.
  • ادغام با ابزارهای NET.: استفاده از Roslyn برای تحلیل وابستگی‌ها و LibGit2Sharp برای بررسی تغییرات Git.

نصب و راه‌اندازی Incrementalist
برای استفاده از Incrementalist، می‌توان آن را به‌صورت ابزار محلی (Local Tool) یا سراسری (Global Tool) نصب کرد. نصب به‌عنوان ابزار سراسری، اجرای دستورات را ساده‌تر می‌کند. مراحل نصب به شرح زیر است:
# ایجاد فایل مانیفست ابزار در مخزن
dotnet new tool-manifest
# نصب Incrementalist به‌صورت جهانی
dotnet tool install Incrementalist.Cmd
پس از نصب، دستورات مستقیماً با incrementalist قابل‌اجرا هستند. برای نصب محلی، از دستور زیر استفاده کنید:
dotnet tool install Incrementalist.Cmd
در این حالت، دستورات با پیشوند dotnet tool run incrementalist اجرا می‌شوند.

استفاده از Incrementalist در فرآیند توسعه
Incrementalist از طریق تحلیل تفاوت‌های Git (Git Diff) بین شاخه‌ها، پروژه‌های تحت تأثیر را شناسایی می‌کند. این ابزار می‌تواند خروجی‌های مختلفی مانند لیست پروژه‌ها یا پوشه‌های تغییر یافته تولید کند که برای اسکریپت‌های ساخت یا خطوط لوله CI/CD مفید است. مثال‌های زیر کاربردهای اصلی آن را نشان می‌دهند:

دریافت لیست پروژه‌های تحت تأثیر:
incrementalist -b dev -f ./affected-projects.txt
این دستور پروژه‌های تغییر یافته نسبت به شاخه dev را در فایل affected-projects.txt ذخیره می‌کند.

ساخت پروژه‌های تحت تأثیر:
incrementalist -b dev -r -- build -c Release --nologo
این دستور پروژه‌های تغییر یافته را با پیکربندی Release می‌سازد.

اجرای تست‌ها با پوشش کد:
incrementalist -b dev -r -- test -c Release --no-build --logger:trx --collect:"XPlat Code Coverage" --results-directory ./testresults
این دستور تست‌های پروژه‌های تحت تأثیر را اجرا کرده و گزارش پوشش کد را تولید می‌کند.

اجرای موازی:
incrementalist -b dev -r --parallel -- build -c Release --nologo
اجرای موازی، زمان ساخت را در سیستم‌های چند‌هسته‌ای کاهش می‌دهد.

مشخص کردن راه‌حل خاص:
incrementalist -s ./src/MySolution.sln -b dev -f ./affected-projects.txt
این دستور تحلیل را به راه‌حل مشخصی محدود می‌کند.

ادغام با خطوط لوله CI/CD
یکی از نقاط قوت Incrementalist، قابلیت ادغام با سیستم‌های CI/CD مانند Azure DevOps یا GitHub Actions است. با ذخیره لیست پروژه‌های تحت تأثیر در یک فایل، می‌توان اسکریپت‌های ساخت را به‌گونه‌ای تنظیم کرد که تنها پروژه‌های لازم را پردازش کنند. برای مثال، در یک خط لوله Azure DevOps، می‌توان از خروجی Incrementalist برای فیلتر کردن پروژه‌ها استفاده کرد:
steps:
- script: incrementalist -b dev -f affected-projects.txt
  displayName: 'Identify affected projects'
- script: dotnet build -c Release --nologo -p:ProjectFile=$(cat affected-projects.txt)
  displayName: 'Build affected projects'
این رویکرد، زمان اجرای خط لوله را به حداقل می‌رساند و منابع محاسباتی را بهینه می‌کند.

مقایسه با سایر ابزارها
ابزارهای دیگری مانند MSBuild یا Cake نیز برای مدیریت فرآیند ساخت استفاده می‌شوند، اما Incrementalist با تمرکز بر تحلیل تغییرات Git و ادغام با Roslyn، رویکردی تخصصی‌تر ارائه می‌دهد. برخلاف MSBuild که کل راه‌حل را می‌سازد، Incrementalist تنها بخش‌های تغییر یافته را هدف قرار می‌دهد، که برای پروژه‌های بزرگ مزیت قابل‌توجهی است.

بهترین شیوه‌ها
برای استفاده بهینه از Incrementalist، موارد زیر توصیه می‌شود:
  • به‌روزرسانی منظم ابزار: برای جلوگیری از باگ‌های شناخته‌شده، همیشه از آخرین نسخه استفاده کنید.
  • استفاده از اجرای موازی: در سیستم‌های قدرتمند، این گزینه زمان پردازش را کاهش می‌دهد.
  • ذخیره خروجی‌ها: فایل‌های خروجی مانند affected-projects.txt برای اتوماسیون و عیب‌یابی مفید هستند.
  • تست در محیط محلی: پیش از ادغام در CI/CD، ابزار را در محیط توسعه تست کنید.

نتیجه‌گیری
Incrementalist ابزاری قدرتمند برای توسعه‌دهندگان NET. است که به دنبال بهینه‌سازی فرآیند ساخت و تست در پروژه‌های پیچیده هستند. با تمرکز بر تغییرات افزایشی، این ابزار زمان اجرای خطوط لوله CI/CD را کاهش داده و بهره‌وری را افزایش می‌دهد. قابلیت‌هایی مانند تحلیل خودکار وابستگی‌ها، اجرای موازی و ادغام با سیستم‌های CI/CD، آن را به گزینه‌ای ایده‌آل برای مخازن تک‌ریشه‌ای و معماری‌های میکروسرویس تبدیل کرده است. با رعایت بهترین شیوه‌ها و به‌روزرسانی منظم، توسعه‌دهندگان می‌توانند از مزایای این ابزار به‌طور کامل بهره‌مند شوند. در نهایت، Incrementalist نه‌تنها یک ابزار فنی، بلکه رویکردی برای مدیریت هوشمند تغییرات در توسعه نرم‌افزار است.
