‫شروع کار با Microsoft Agent Framework: ساخت چت‌بات پشتیبانی با C#


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۵ ۱۰:۴۴
آدرس: www.dntips.ir
مایکروسافت سال‌هاست که روی ابزارهایی برای ساخت برنامه‌های کاربردی با هوش مصنوعی سرمایه‌گذاری می‌کند. ابتدا Semantic Kernel و سپس AutoGen معرفی شدند و اکنون هر دو در Microsoft Agent Framework (MAF) ادغام شده‌اند. MAF بهترین ویژگی‌های هر دو را ترکیب می‌کند – پایداری و ویژگی‌های سازمانی Semantic Kernel با انتزاع‌های ساده برای Agentهای چند مرحله‌ای AutoGen – و قابلیت‌های جدیدی مانند گردش‌کارهای مبتنی بر گراف و یک سیستم مدیریت حالت قوی را اضافه می‌کند. در زمان نگارش این مقاله، MAF در حالت پیش‌نمایش عمومی (public preview) قرار دارد و با مجوز MIT برای .NET و Python در دسترس است. این فریم‌ورک حول دو مفهوم اصلی می‌چرخد: Agentها و Workflows. Agentها سیستم‌هایی هستند که از یک مدل زبانی بزرگ (LLM) برای پردازش ورودی‌ها، تصمیم‌گیری، فراخوانی ابزارها و تولید پاسخ استفاده می‌کنند. Workflows دنباله‌ای از مراحل تعریف‌شده هستند که در یک گراف به هم متصل شده‌اند. در این مقاله، مفاهیم اساسی فریم‌ورک را یاد خواهیم گرفت و یک API با ASP.NET Core ایجاد می‌کنیم که یک چت‌بات پشتیبانی فنی را در معرض دید قرار می‌دهد. این چت‌بات قادر خواهد بود به سؤالات بر اساس مستندات Markdown داخلی پاسخ دهد و در عین حال، زمینه مکالمه را بین پیام‌ها حفظ کند. کد کامل این پروژه در GitHub در دسترس است. MAF بر اساس دو مفهوم کلیدی به نام‌های Agent و Workflow بنا شده است. Agentها از LLM برای پردازش ورودی‌ها و تصمیم‌گیری استفاده می‌کنند، در حالی که Workflowها توالی مراحل تعریف‌شده‌ای هستند. نکته مهم این است که اگر می‌توان یک تابع برای انجام یک کار نوشت، بهتر است از Agent استفاده نشود. پنج مفهوم کلیدی در MAF وجود دارد: ایجاد اولین Agent با استفاده از یک کلاینت چت (مانند Azure OpenAI)، تعریف Tools با استفاده از ویژگی [Description] و AIFunctionFactory.Create()، مدیریت مکالمات چند مرحله‌ای با استفاده از AgentSession، مدیریت حافظه و پایداری با سریال‌سازی AgentSession به JsonElement.
