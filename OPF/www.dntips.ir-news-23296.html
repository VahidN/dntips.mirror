‫ایجاد کتابخانه درخواست (Prompt Library) برای توسعه‌دهندگان Vibe Coding


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۰ ۱۳:۰۵
آدرس: www.dntips.ir
در روز بیست و هفتم از چالش Vibe Coding، جف بلنکنبرگ بر اهمیت ایجاد و نگهداری یک کتابخانه درخواست (Prompt Library) برای توسعه‌دهندگان تاکید می‌کند. این کتابخانه به عنوان یک سرمایه‌گذاری در زمان و بهره‌وری عمل می‌کند و به توسعه‌دهندگان کمک می‌کند تا از درخواست‌های قبلی که به خوبی کار کرده‌اند، مجدداً استفاده کنند و از تکرار آن‌ها جلوگیری کنند. ایجاد یک کتابخانه درخواست شامل جمع‌آوری درخواست‌هایی است که بیش از یک بار استفاده شده‌اند، زمان زیادی برای تنظیم آن‌ها صرف شده است، به طور مداوم به خوبی کار می‌کنند و برای وظایف رایج قابل استفاده هستند. درخواست‌هایی که برای موقعیت‌های خاص و یک‌بار استفاده می‌شوند، درخواست‌هایی که کار نکرده‌اند و درخواست‌هایی که فقط شامل متن پروژه هستند، نباید در این کتابخانه ذخیره شوند. ساختار کتابخانه درخواست پیشنهادی بر اساس نوع وظیفه سازماندهی شده است، شامل دسته‌بندی‌هایی مانند تولید کد، بررسی کد، اشکال‌زدایی، بازسازی کد و استقرار. هر فایل شامل یک قالب درخواست با مکان‌نماها، یک مثال و تغییرات مختلف است. قالب‌های درخواست شامل بخش‌هایی برای شرح استفاده، الگو، مثال و تغییرات هستند که به توسعه‌دهندگان کمک می‌کند تا به راحتی درخواست‌های خود را سفارشی‌سازی کنند. جف بلنکنبرگ یک مجموعه شروع (Starter Kit) برای کتابخانه درخواست ارائه داده است که شامل الگوهای آماده برای وظایف مختلف توسعه نرم‌افزار است. این مجموعه شامل الگوهایی برای تولید API Endpoint، Feature Generation، Test Generation و Code Review است که به توسعه‌دهندگان کمک می‌کند تا به سرعت یک کتابخانه درخواست کارآمد ایجاد کنند و بهره‌وری خود را افزایش دهند.
