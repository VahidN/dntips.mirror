‫ویرایش‌های GitHub Copilot: راهنمای توسعه‌دهنده‌ها برای یک پایگاه کد تمیزتر با کمک عوامل هوش مصنوعی


نویسنده: وحید نصیری
تاریخ: ۱۴۰۳/۱۲/۰۳ ۰۷:۳۱
آدرس: www.dntips.ir
این مقاله بررسی می‌کند که چگونه می‌توان از ویرایش‌های GitHub Copilot (GCE) برای تمیز کردن و بازسازی پایگاه‌های کد CSS به طور مؤثر استفاده کرد. GCE از هوش مصنوعی برای ایجاد تغییرات جامع کد در چندین فایل بر اساس دستورالعمل‌های زبان طبیعی استفاده می‌کند.

ویرایش‌های GitHub Copilot چیست؟

  • یک ویژگی در VS Code که از هوش مصنوعی برای تغییرات کد استفاده می‌کند.
  • در دو حالت عمل می‌کند:
    • حالت ویرایش: کاربر فایل‌ها را انتخاب می‌کند و یک درخواست را ارائه می‌دهد.
    • حالت عامل: Copilot به طور خودکار فایل‌های نیازمند تغییر را تعیین می‌کند و دستورات ترمینال را پیشنهاد می‌دهد.

موارد استفاده کلیدی برای تمیز کردن CSS با GCE:

    • حذف کلاس‌های CSS استفاده نشده: درخواست: "کلاس‌های CSSای را که اصلا در فایل‌های TSX استفاده نشده‌اند، حذف کن"
    • GCE فایل‌های CSS و TSX را تجزیه و تحلیل می‌کند، کلاس‌های استفاده نشده را شناسایی می‌کند و حذف آنها را پیشنهاد می‌دهد.
    • مدیریت بهتر رنگ با متغیرهای CSS: درخواست: "#codebase همه مقادیر رنگ هگز تکراری در پایگاه کد را بگیر و آنها را با یک نام معقول به متغیرهای ریشه اضافه کن"
    • GCE پایگاه کد را اسکن می‌کند، متغیرهای CSS را برای رنگ‌های هگز تکراری ایجاد می‌کند و موارد استفاده را با این متغیرها جایگزین می‌کند.
    • سازماندهی مجدد فایل‌ها برای ساختار بهتر: درخواست: "این پوشه را به پوشه‌های فرعی سازماندهی کن و imports را تنظیم کن"
    • GCE یک ساختار پوشه منطقی را پیشنهاد می‌دهد، فایل‌ها را منتقل می‌کند و عبارات imports را به روز می‌کند.
    • کار با کل پایگاه کد: درخواست: "#codebase همه CSS را به استفاده از یک رویکرد موبایل-اول با پرس و جوهای رسانه تبدیل کن"
    • GCE می‌تواند تغییرات را در کل پایگاه کد اعمال کند.

بهترین روش‌ها:

  • در درخواست‌ها مشخص باشید.
  • تغییرات را با دقت بررسی کنید.
  • برای وظایف پیچیده از حالت عامل استفاده کنید.
  • قبل از تغییرات بزرگ، کامیت کنید.
