‫آزمایش GitHub Copilot برای .NET، تست‌های واحد مبتنی بر هوش مصنوعی را به Visual Studio 2026 می‌آورد


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۳ ۰۱:۴۳
آدرس: www.dntips.ir
آزمایش GitHub Copilot برای .NET اکنون به طور کلی در Visual Studio 2026 نسخه 18.3 در دسترس است و فرآیند تولید تست‌های واحد با کیفیت را برای توسعه‌دهندگان به طور چشمگیری سریع‌تر و آسان‌تر می‌کند. این نسخه، یکپارچگی بهتری با IDE، دستورات طبیعی‌تر و نقاط ورود جدیدی را ارائه می‌دهد که تولید تست‌های واحد را بدون نیاز به ترک کد، آسان و شهودی می‌سازد. این ویژگی به طور خاص برای تست واحد طراحی شده است و از ساختار راه حل، فریم‌ورک‌های تست و سیستم ساخت شما آگاه است. GitHub Copilot به عنوان یک گردش کار تست سرتاسری عمل می‌کند، نه یک پاسخ واحد به یک درخواست. شما می‌توانید تست‌ها را در مقیاسی که برای وظیفه شما مناسب است، تولید کنید، خواه یک عضو، یک کلاس، یک فایل، کل پروژه، یک راه حل کامل یا تفاوت‌های فعلی git باشد. هنگامی که یک درخواست تست را شروع می‌کنید، GitHub Copilot تست‌های واحدی را در محدوده کد انتخاب شده تولید می‌کند، آن‌ها را به طور خودکار ساخت و اجرا می‌کند، شکست‌ها را شناسایی و سعی در رفع آن‌ها می‌کند و تا زمانی که یک نقطه شروع پایدار داشته باشید، دوباره اجرا می‌کند. این عامل همچنین با هر مدلی که در Copilot Chat انتخاب کرده‌اید کار می‌کند و به شما امکان می‌دهد از مدل ترجیحی خود در حین استفاده از گردش کار تست اختصاصی بهره‌مند شوید. پس از اتمام تولید تست، Copilot خلاصه‌ای ساختاریافته ارائه می‌دهد که به شما کمک می‌کند تا درک کنید چه تغییراتی ایجاد شده است: فایل‌ها و پروژه‌های تست ایجاد یا اصلاح شده، اطلاعات پوشش قبل و بعد، سیگنال‌های موفقیت/شکست و موارد ناپایدار و بینش‌هایی در مورد شکاف‌های قابلیت آزمایش. همچنین پیوندهای مستقیمی به تست‌های تولید شده وجود دارد تا بتوانید بلافاصله آن‌ها را بررسی و تکرار کنید.
