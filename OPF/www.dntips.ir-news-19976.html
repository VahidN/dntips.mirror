‫خلاصه تجربه استفاده از GitHub Copilot رایگان


نویسنده: وحید نصیری
تاریخ: ۱۴۰۳/۱۱/۲۱ ۰۸:۰۲
آدرس: www.dntips.ir
این مقاله تجربه نویسنده را در استفاده از نسخه رایگان GitHub Copilot خلاصه می‌کند.

نمای کلی طرح رایگان

  • 2000 تکمیل کد و 50 درخواست چت در ماه ارائه می‌دهد.
  • شامل دسترسی به مدل‌های هوش مصنوعی مانند Claude 3.5 Sonnet، Gemini 2.0 Flash، o3-mini و GPT-4o است.
  • برای کدنویسی معمولی و پروژه‌های جانبی مناسب است، اما احتمالاً برای کارهای سنگین یا حرفه‌ای کافی نیست.

تنظیم و ادغام

  • به طور یکپارچه در IDEها (VS Code در مثال استفاده شده) ادغام می‌شود.
  • فرآیند ثبت‌نام ساده در داخل IDE.
  • پیکربندی حداقلی مورد نیاز است.

آزمایش و مشاهدات

  • با یک پروژه سرگرمی Next.js (برنامه رستوران‌یاب) آزمایش شد.

تکمیل کد

  • وارد کردن‌ها: برای فایل‌های موجود عملکرد خوبی داشت و از قالب‌بندی ویرایشگر پیروی می‌کرد. در فایل‌های خالی در مورد نقطه‌ویرگول‌ها ناهماهنگی‌هایی وجود داشت.
  • داده‌های آزمایشی: به طور مؤثر داده‌های آزمایشی ساختاریافته را بر اساس انواع تعریف‌شده تولید کرد.
  • بازسازی: بازسازی را با استفاده از یک کامپوننت سفارشی (<Badge />) پیشنهاد کرد، اما با ویژگی‌ها مشکل داشت.
  • ساخت کامپوننت: ابتدا با پیشنهادات ویژگی برای یک کامپوننت KeyboardShortcut مشکل داشت، اما با تعریف صریح ویژگی‌ها بهبود یافت.

چت Copilot

  • بازسازی با classnames: با یک درخواست ساده، با موفقیت یک کامپوننت را با استفاده از کتابخانه classnames بازسازی کرد. زمینه فایل فعلی، به طور خودکار در چت گنجانده می‌شود.
  • عیب‌یابی مسیر API: هنگام ارائه زمینه دقیق و اطلاعات دقیق خطا از طریق دستورات slash “/fix”، یک خطا را در یک مسیر API در Next.js شناسایی و برطرف کرد. درخواست‌های ساده‌تر، پاسخ‌های نامربوطی را به همراه داشت.

نتیجه‌گیری و توصیه‌ها

  • برای کارهای تکراری و تولید کد اولیه مفید است.
  • بدون زمینه کافی می‌تواند با قالب‌بندی و منطق پیچیده مشکل داشته باشد.
  • ادغام یکپارچه با IDE یک مزیت قابل توجه است.
  • کیفیت پاسخ به شدت به وضوح و جزئیات درخواست بستگی دارد.
  • برای پروژه‌های شخصی و کوچک توصیه می‌شود.
  • بررسی دستی، برای کد تولیدی، همچنان ضروری است.
  • درخواست‌های دقیق برای به حداکثر رساندن اثربخشی Copilot حیاتی هستند.
