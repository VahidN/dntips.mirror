‫روش نصب NET SDK. بر روی لینوکس Ubuntu


نویسنده: وحید نصیری
تاریخ: ۱۴۰۳/۰۵/۱۹ ۱۵:۵۵
آدرس: www.dntips.ir

اگر از آخرین نگارش Ubuntu استفاده می‌کنید، با توجه به همکاری مایکروسافت و شرکت پشتیبان آن، نصب دات‌نت به سادگی اجرای دستور زیر است:

$ sudo apt update && sudo apt install -y dotnet-sdk-8.0
$ dotnet --version

و اگر می‌خواهید بدانید که چه ‌نگارشی از دات‌نت، به‌همراه مخازن استاندارد Ubuntu است، دستور زیر را صادر کنید:

$ apt search dotnet-sdk*

که یک نمونه خروجی آن به صورت زیر است:

$ apt search dotnet-sdk*
Sorting... Done

نصب اسکریپتی آخرین نگارش دات‌نت بر روی تمام توزیع‌های لینوکسی

مایکروسافت، اسکریپتی را برای دریافت و نصب خودکار آخرین نگارش دات‌نت، تهیه کرده‌است که کار با آن بسیار ساده‌است و با تمام توزیع‌های لینوکسی و نه فقط Ubuntu سازگار است.

پیش از هرکاری ابتدا مخزن‌های بسته‌ها و برنامه‌های مرتبط را یکبار به‌روز کرده و سیستم را ری‌استارت می‌کنیم:

sudo apt update -q && sudo apt upgrade -y && reboot

سپس دستور زیر را صادر می‌کنیم تا اسکریپت نصاب مخصوص دات‌نت خود مایکروسافت را دریافت کنیم :

wget https://dot.net/v1/dotnet-install.sh -O dotnet-install.sh
chmod +x ./dotnet-install.sh

توسط این دو دستور، فایل dotnet-install.sh دریافت شده و همچنین دسترسی اجرایی بودن آن نیز تنظیم می‌شود.

پس از آن، برای نصب آخرین نگارش دات‌نت SDK موجود، تنها کافی است دستور زیر را صادر کنیم:

./dotnet-install.sh --version latest

و یا اگر فقط می‌خواهید runtime آن‌را نصب کنید، پارامترهای نصب، به صورت زیر تغییر می‌کنند:

./dotnet-install.sh --version latest --runtime aspnetcore

همچنین اگر نگارش‌های پایین‌تر مدنظر شما هستند، می‌توانید کانال نصب را هم مشخص کنید:

./dotnet-install.sh --channel 7.0

که یک نمونه خروجی اجرای دستور dotnet-install.sh --version latest/. آن به صورت زیر است:

$ ./dotnet-install.sh --version latest

همانطور که مشاهده می‌کنید، دات‌نت 8.0.303، نصب شده‌است.

پس از پایان نصب، دو دستور زیر را هم باید اجرا کنید تا بتوان در خط فرمان به NET CLI. دسترسی یافت:

$ export DOTNET_ROOT=$HOME/.dotnet
$ export PATH=$PATH:$DOTNET_ROOT:$DOTNET_ROOT/tools

پس از این تنظیمات، امکان اجرای دو دستور آزمایشی زیر میسر می‌شوند:

$ dotnet --version
$ dotnet --list-sdks

نصب دات‌نت بر روی نگارش‌های قدیمی‌تر Ubuntu

برای اینکه بتوان به روش متداولی (بدون دریافت اسکریپت نصاب فوق) به آخرین تغییرات انجام شده و آخرین به‌روز رسانی‌ها و همچنین تمام نگارش‌های دات‌نت دسترسی داشت، می‌توان از مخازن بسته‌های خود مایکروسافت استفاده کنیم که باید آن‌ها را به سیستم عامل اضافه کرد. برای اینکار باید مراحل زیر طی شوند:

ابتدا تمام وابستگی‌های احتمالی موجود را حذف می‌کنیم:

$ sudo apt remove 'dotnet*' 'aspnet*' 'netstandard*'

سپس فایل زیر را ایجاد کرده:

sudo nano touch /etc/apt/preferences

و آن‌را با محتوای زیر تکمیل می‌کنیم؛ تا فقط از مخازن خود مایکروسافت استفاده شود و سایر مخازن مرتبط در این حالت، اولویت کمتری داشته باشند:

Package: dotnet* aspnet* netstandard*
Pin: origin "archive.ubuntu.com"
Pin-Priority: -10

Package: dotnet* aspnet* netstandard*
Pin: origin "security.ubuntu.com"
Pin-Priority: -10

پس از آن، دستورات زیر، کار افزودن مخازن بسته‌های مایکروسافت را انجام می‌دهند:

# Get OS version info which adds the $ID and $VERSION_ID variables
source /etc/os-release

# Download Microsoft signing key and repository
wget https://packages.microsoft.com/config/$ID/$VERSION_ID/packages-microsoft-prod.deb -O packages-microsoft-prod.deb

# Install Microsoft signing key and repository
sudo dpkg -i packages-microsoft-prod.deb

# Clean up
rm packages-microsoft-prod.deb

# Update packages
sudo apt update
sudo apt upgrade -y

بعد از این به‌روز رسانی‌ها، دستور متداول زیر، کار نصب آخرین نگارش NET SDK. را انجام می‌دهد:

sudo apt-get install -y dotnet-sdk-8.0

و همچنین هربار هم که سیستم را با دستورات sudo apt update -q && sudo apt upgrade -y به روز کنیم، در صورت وجود به‌روز رسانی دات‌نتی جدیدی، آن‌را به صورت خودکار دریافت و نصب می‌کند.

    نصب و به‌روزرسانی .NET 10 روی اوبونتو (و WSL)

    هر سال با انتشار نسخه جدید .NET، نیاز داریم SDK را روی ماشین‌هایمان به‌روز کنیم؛ نه فقط روی ویندوز، بلکه روی توزیع‌های لینوکس مثل اوبونتو که داخل WSL یا مستقیم استفاده می‌کنیم. حالا که .NET 10 (نسخه LTS با پشتیبانی ۳ ساله تا نوامبر ۲۰۲۸) منتشر شده، بیایید ببینیم چطور می‌توانیم آن را به ساده‌ترین و مطمئن‌ترین شکل ممکن نصب یا ارتقا دهیم.
    در این مقاله، تمرکزمان روی روشی است که همیشه کار می‌کند: استفاده از اسکریپت رسمی dotnet-install.sh از مایکروسافت. این روش حتی وقتی بسته‌های APT هنوز به‌روز نشده باشند (که گاهی چند روز یا هفته طول می‌کشد) هم فوراً نسخه جدید را در اختیارتان قرار می‌دهد و کاملاً قابل اعتماد است.

    چرا apt-get گاهی کار نمی‌کند؟
    اگر مستقیم این دستورها را بزنید:
    sudo apt update
sudo apt install dotnet-sdk-10.0
    ممکن است با خطای «Unable to locate package dotnet-sdk-10.0» مواجه شوید. دلیلش این است که بسته‌های .NET در مخازن اوبونتو/دبیان با کمی تأخیر ظاهر می‌شوند (به‌خصوص در روزهای اول انتشار). روش اسکریپت اما مستقیم از سرورهای مایکروسافت دانلود می‌کند و همیشه آخرین نسخه را دارد.

    روش پیشنهادی: نصب با dotnet-install.sh

    ۱. اسکریپت را دانلود و اجرایی کنید:
    wget https://dot.net/v1/dotnet-install.sh -O dotnet-install.sh
chmod +x dotnet-install.sh
    نکته امنیتی مهم: حتماً فایل را قبل از اجرا بازبینی کنید! می‌توانید با cat dotnet-install.sh یا less dotnet-install.sh محتوای آن را ببینید. این اسکریپت رسمی مایکروسافت است، اما عادت خوب این است که هیچ اسکریپتی را کورکورانه اجرا نکنید.

    ۲. نسخه دلخواه را نصب کنید (برای آخرین نسخه کانال ۱۰.۰):
    ./dotnet-install.sh --channel 10.0
    خروجی چیزی شبیه این خواهد بود:
    dotnet-install: Installed version is 10.0.100
dotnet-install: Adding to current process PATH: `/home/youruser/.dotnet`
dotnet-install: Installation finished successfully.
    اگر می‌خواهید نسخه دقیق (مثلاً ۱۰.۰.۱۰۰) را نصب کنید:
    ./dotnet-install.sh --version 10.0.100

    ۳. بررسی نصب:
    dotnet --version          # باید 10.0.100 یا جدیدتر نشان دهد
dotnet --list-sdks        # لیست همه SDKهای نصب‌شده

    تنظیم PATH دائمی (تا همیشه نسخه جدید اولویت داشته باشد)
    به‌طور پیش‌فرض، SDK در $HOME/.dotnet نصب می‌شود. برای اینکه همیشه این نسخه (کاربرمحور) نسبت به نسخه‌های سیستمی (/usr/share/dotnet یا مشابه) اولویت داشته باشد، این خطوط را به فایل ~/.bashrc یا ~/.zshrc اضافه کنید:
    export DOTNET_ROOT="$HOME/.dotnet"
export PATH="$DOTNET_ROOT:$DOTNET_ROOT/tools:$PATH"
    سپس:
    source ~/.bashrc   # یا shell خودتان را ری‌استارت کنید
    حالا بررسی کنید:
    which dotnet      # باید به ~/.dotnet/dotnet اشاره کند
dotnet --version

    مشکلات رایج و راه‌حل‌ها
    مشکلدلیل احتمالیراه‌حل
    هنوز نسخه قدیمی نشان می‌دهدPATH اشتباه یا global.json قدیمیwhich dotnet را چک کنید و PATH را اصلاح کنید
    پروژه روی نسخه قدیمی قفل شدهوجود فایل قدیمی global.json در پوشه پروژه یا والدجستجو کنید: find . -name global.json فایل را حذف یا نسخه را به‌روز کنید

    نکته‌های تکمیلی
    • اگر چندین نسخه SDK همزمان نیاز دارید (که معمولاً همین‌طور است)، این روش عالی کار می‌کند چون همه نسخه‌ها کنار هم در ~/.dotnet نصب می‌شوند و با global.json می‌توانید برای هر پروژه نسخه خاص را مشخص کنید.
    • برای محیط‌های سرور یا CI/CD، می‌توانید از پرچم --install-dir /opt/dotnet استفاده کنید تا نصب سیستمی شود.
    • وابستگی‌ها (dependencies): اسکریپت، وابستگی‌های سیستم را نصب نمی‌کند. برای لیست کامل به صفحه رسمی مایکروسافت مراجعه کنید: https://learn.microsoft.com/dotnet/core/install/linux-ubuntu

    نتیجه‌گیری
    با این روش ساده و مستقیم، دیگر لازم نیست منتظر به‌روزرسانی مخازن apt بمانید. فقط چند دستور و .NET 10 آماده استفاده است؛ چه روی WSL، چه روی سرور اوبونتو، چه در ماشین مجازی. این رویکرد همیشه به‌روز، امن و کاملاً تحت کنترل شماست.
