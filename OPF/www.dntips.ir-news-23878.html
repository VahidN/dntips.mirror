‫ساخت یک عامل تحقیق هوش مصنوعی محلی در C# - از صفر تا تحقیق خودکار


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۰۹:۵۰
آدرس: www.dntips.ir
یک توسعه‌دهنده روشی برای ساخت یک عامل تحقیق هوش مصنوعی محلی با استفاده از C# و .NET 8 را شرح داده است. این عامل، Axiom نام دارد، و می‌تواند به طور خودکار وب را جستجو کند، صفحات را بخواند و گزارش‌های تحقیقاتی تولید کند، همه این کارها بدون نیاز به کلیدهای API ابری و به صورت محلی انجام می‌شود. Axiom از Brave Search API برای جستجو در وب، SQLite برای ذخیره‌سازی حافظه و جستجوی معنایی و Ollama برای استنتاج LLM (مدل زبان بزرگ) استفاده می‌کند. معماری آن شامل تولید پرس و جوهای جستجو، جستجو در Brave API، بازیابی و تجزیه و تحلیل صفحات وب و در نهایت، ترکیب یافته‌ها در یک گزارش ساختاریافته است. این فرآیند به صورت تکراری انجام می‌شود و گزارش‌ها به صورت Markdown ذخیره می‌شوند. این پروژه از C# به جای Python انتخاب شده است، زیرا C# ابزارهای بهتری (مانند Visual Studio و Rider)، استقرار آسان‌تر (یک فایل اجرایی واحد) و عملکرد سریع‌تری (زمان شروع سریع‌تر و حافظه کمتر) ارائه می‌دهد. این عامل از یک سیستم ابزار (Tool System) استفاده می‌کند که در آن هر قابلیت به عنوان یک ITool تعریف می‌شود و LLM تصمیم می‌گیرد کدام ابزارها را فراخوانی کند. برای مدیریت حافظه، به جای پایگاه داده برداری (مانند ChromaDB یا FAISS)، از SQLite با ذخیره کردن امبدینگ‌ها (embeddings) به صورت BLOB استفاده شده است. این روش برای داده‌های در مقیاس شخصی بسیار کارآمد است. ResearchRunner، حالت تحقیق خودکار، کل فرآیند را از تولید پرس و جو تا ترکیب یافته‌ها، هماهنگ می‌کند. این پروژه یک راه حل جذاب برای توسعه‌دهندگان .NET ارائه می‌دهد که به دنبال ابزارهای هوش مصنوعی هستند و می‌خواهند از قدرت مدل‌های زبان بزرگ به صورت محلی و بدون وابستگی به سرویس‌های ابری استفاده کنند. این رویکرد به ویژه برای کسانی که به دنبال حفظ حریم خصوصی و کنترل داده‌های خود هستند، جذاب است.
