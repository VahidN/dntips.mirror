‫ساخت یک عامل درخواست کشش مبتنی بر هوش مصنوعی برای Azure DevOps با استفاده از GitHub Copilot SDK


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۲/۰۶ ۰۱:۱۳
آدرس: www.dntips.ir
توسعه‌دهندگان اکنون می‌توانند از یک عامل درخواست کشش مبتنی بر هوش مصنوعی برای Azure DevOps بهره‌مند شوند، حتی اگر کد آن‌ها در GitHub نباشد. این عامل، که با استفاده از GitHub Copilot SDK ساخته شده است، به طور خودکار درخواست‌های کشش را در Azure DevOps بررسی می‌کند و گزارش‌های Markdown ساختاریافته‌ای را در مورد آسیب‌پذیری‌های امنیتی، گلوگاه‌های عملکرد، نگرانی‌های مربوط به قابلیت نگهداری و پیشنهادات اصلاح ارائه می‌دهد. این عامل با استفاده از GitHub Copilot CLI برای اجرای وظایف ارتباط برقرار می‌کند و از دو سرور MCP (Model Context Protocol) استفاده می‌کند: یکی برای بازیابی جزئیات درخواست کشش از Azure DevOps و دیگری برای جستجوی مستندات بهترین شیوه‌ها از Microsoft Learn. این رویکرد تضمین می‌کند که بررسی کد بر اساس راهنمایی‌های رسمی و به‌روز انجام می‌شود. عملکرد اصلی این عامل شامل دریافت تفاوت‌های PR، تجزیه و تحلیل کد با استفاده از یک مدل هوش مصنوعی (مانند Claude Sonnet) و تولید یک گزارش Markdown جامع است. این گزارش به توسعه‌دهندگان کمک می‌کند تا مشکلات بالقوه را قبل از ادغام کد شناسایی و رفع کنند، در نتیجه کیفیت کد را بهبود می‌بخشند. این راه حل به صورت کانتینر بسته‌بندی شده است که توزیع و اجرای آن را در تیم‌ها و خطوط لوله مختلف آسان می‌کند. استفاده از Azure Container Registry (ACR) برای میزبانی تصاویر، امنیت و دسترسی را افزایش می‌دهد و از ویژگی‌های امنیتی ACR مانند امضای تصویر و اسکن آسیب‌پذیری بهره می‌برد. این ابزار به تیم‌هایی که هنوز نمی‌توانند کد خود را به GitHub منتقل کنند، امکان می‌دهد از مزایای بررسی کد مبتنی بر هوش مصنوعی بهره‌مند شوند و سطح نوآوری و مراقبت را در فرآیند DevOps خود افزایش دهند.
