‫توسعه‌ی یک بازی کارتی با استفاده از Claude و Vibe Coding


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۵ ۰۱:۲۳
آدرس: www.dntips.ir
نویسنده، جف بلنکنبرگ، در سری مقالات ۳۱ روزه Vibe Coding، تجربه‌ی ساخت یک بازی کارتی به نام Backalley را با استفاده از Claude شرح می‌دهد. این بازی با یک دسته‌ی کارت استاندارد ۵۲ تایی انجام می‌شود و شامل ۲۰ دور بازی است که در هر دور تعداد کارت‌های توزیع شده بین بازیکنان از ۱۰ تا ۱ کاهش می‌یابد و سپس دوباره به ۱۰ می‌رسد. هدف از این پروژه، ساخت یک سیستم امتیازدهی دیجیتال برای این بازی بود که به راحتی قابل استفاده باشد. نویسنده از Claude برای تفسیر یک تصویر دست‌نویس از صفحه‌ی امتیازدهی بازی استفاده کرد و Claude توانست به خوبی اجزای مختلف تصویر را شناسایی کرده و از آن به عنوان الگو برای صفحه‌ی دیجیتال استفاده کند. این تجربه نشان می‌دهد که می‌توان از تصاویر برای توصیف رابط کاربری استفاده کرد، به خصوص زمانی که کلمات به تنهایی کافی نیستند. در این پروژه، نویسنده از dexie.js برای ذخیره‌سازی داده‌ها در مرورگر استفاده کرد، که امکان جمع‌آوری داده‌ها بین کاربران و بازی‌ها را فراهم نمی‌کرد. نویسنده همچنین از Supabase برای ساخت یک سیستم احراز هویت مبتنی بر ایمیل با استفاده از لینک جادویی و دامنه سفارشی استفاده کرد. یکی دیگر از ویژگی‌های جالب این برنامه، دکمه‌ی "اعلام رتبه‌بندی" است که با استفاده از فناوری تبدیل متن به گفتار، رتبه‌ی فعلی هر بازیکن را اعلام می‌کند. این پروژه در مخازن GitHub نویسنده قابل دسترسی است. این پروژه نشان‌دهنده‌ی استفاده از ابزارهای مختلف در فرآیند Vibe Coding است، از جمله Claude برای تفسیر تصاویر، dexie.js برای ذخیره‌سازی داده‌ها در مرورگر و Supabase برای احراز هویت. نویسنده تاکید می‌کند که این یک سری مقالات رایگان است که بر اساس یادگیری‌های واقعی از صدها ساعت کدنویسی Vibe Coding است.
