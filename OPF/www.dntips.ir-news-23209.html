‫بررسی نحوه Checkout کردن یک مخزن Git با استفاده از Tag در VSCode


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۰۹ ۱۶:۴۲
آدرس: www.dntips.ir
این مقاله راهنمایی می‌کند که چگونه یک مخزن Git را با استفاده از یک Tag در Visual Studio Code Checkout کنید. Tagها در Git به عنوان نقاط خاصی در تاریخچه مخزن عمل می‌کنند و برای علامت‌گذاری نقاط انتشار (مانند v1.0.0، v2.1.3) استفاده می‌شوند. برخلاف شاخه‌ها، Tagها به عنوان نمونه‌های غیرقابل تغییر از کد در یک لحظه خاص در نظر گرفته می‌شوند. برای Checkout کردن یک Tag در VSCode، دو روش اصلی وجود دارد: استفاده از Command Palette و استفاده از Source Control Panel. روش Command Palette سریع‌تر است و با فشردن Ctrl+Shift+P (یا Cmd+Shift+P در macOS) و جستجوی "Git: Checkout to..." قابل دسترسی است. سپس می‌توانید Tag مورد نظر خود را از لیست انتخاب کنید. روش دیگر از طریق Source Control Panel در VSCode است که با کلیک بر روی آیکون Source Control و سپس انتخاب Tag از لیست refs قابل انجام است. هنگامی که یک Tag را Checkout می‌کنید، Git به شما اطلاع می‌دهد که در یک حالت "detached HEAD" قرار دارید. این بدان معناست که شما در حال مشاهده مخزن در یک نقطه زمانی خاص هستید و به یک شاخه متصل نیستید. در این حالت، می‌توانید کد را مرور کنید، Buildها را اجرا کنید و نسخه Tag شده را تست کنید. با این حال، اگر تغییراتی ایجاد کرده و آن‌ها را Commit کنید، این Commitها به هیچ شاخه‌ای تعلق نخواهند گرفت. برای جلوگیری از این مشکل، توصیه می‌شود یک شاخه جدید ایجاد کنید. این کار را می‌توان با استفاده از دستور `git checkout -b new-branch-name` انجام داد. این دستور یک شاخه جدید ایجاد می‌کند و شما را به آن شاخه منتقل می‌کند، به طوری که Commitهای جدید به آن شاخه تعلق می‌گیرند. این روش به شما امکان می‌دهد تا تغییرات خود را ذخیره کنید و آن‌ها را با دیگران به اشتراک بگذارید.
