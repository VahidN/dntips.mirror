‫توسعه یک کامپایلر C با استفاده از تیم‌های موازی Claude


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۲۲ ۰۹:۴۳
آدرس: www.dntips.ir
محققان در Anthropic یک رویکرد جدید برای نظارت بر مدل‌های زبانی با استفاده از "تیم‌های عامل" را آزمایش کرده‌اند. این تیم‌ها شامل چندین نمونه از Claude هستند که به صورت موازی بر روی یک کد منبع مشترک کار می‌کنند بدون نیاز به مداخله مستقیم انسان. هدف از این آزمایش، ارزیابی پتانسیل توسعه دهندگان نرم‌افزاری خودکار است. تیم 16 عاملی با استفاده از Opus 4.6 وظیفه ساخت یک کامپایلر C مبتنی بر Rust را از ابتدا بر عهده گرفت که قادر به کامپایل کرنل لینوکس باشد. این پروژه که شامل تقریباً 2000 جلسه Claude Code و هزینه‌ای معادل 20000 دلار برای API بود، منجر به تولید یک کامپایلر 100000 خطی شد که می‌تواند Linux 6.9 را بر روی معماری‌های x86، ARM و RISC-V کامپایل کند. این کامپایلر به خودی خود یک دستاورد جالب است، اما تمرکز اصلی بر روی درس‌هایی است که در مورد طراحی سیستم‌های harness برای تیم‌های عامل خودکار و طولانی‌مدت آموخته شده است. یکی از چالش‌های اصلی، ایجاد آزمون‌هایی است که عامل‌ها را در مسیر درست نگه دارند بدون نیاز به نظارت انسانی. همچنین، ساختاردهی کار به گونه‌ای که چندین عامل بتوانند به صورت موازی پیشرفت کنند، اهمیت دارد. رویکرد مورد استفاده شامل قرار دادن Claude در یک حلقه ساده است (مشابه Ralph-loop) که پس از اتمام یک وظیفه، بلافاصله وظیفه بعدی را انتخاب می‌کند. این حلقه به طور مداوم اجرا می‌شود و Claude را تشویق می‌کند تا تا زمانی که به نتیجه مطلوب نرسد، به کار خود ادامه دهد. برای بهره‌گیری از قدرت پردازشی موازی، پیاده‌سازی از چندین کانتینر Docker استفاده می‌کند که هر کدام یک نسخه از کد را در یک فضای کاری جداگانه کپی می‌کنند. یک الگوریتم همگام‌سازی ساده برای جلوگیری از تلاش همزمان چندین عامل برای حل یک مسئله واحد به کار گرفته شده است. این رویکرد نشان می‌دهد که چگونه می‌توان با استفاده از مدل‌های زبانی بزرگ، فرآیندهای توسعه نرم‌افزار را خودکار کرد و به نتایج قابل توجهی دست یافت.
