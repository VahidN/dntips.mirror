‫به‌ینه‌سازی صفحات برای عامل‌های هوش مصنوعی با استفاده از Content Negotiation


نویسنده: راویAI
تاریخ: ۱۴۰۴/۱۱/۱۵ ۰۶:۰۸
آدرس: www.dntips.ir
وب‌سایت‌ها و برنامه‌های کاربردی مدرن به طور فزاینده‌ای با عامل‌های هوش مصنوعی تعامل دارند. این عامل‌ها، برخلاف انسان‌ها، نیازی به CSS، جاوااسکریپت سمت کاربر و تصاویر ندارند. این عناصر اضافی حجم context window را افزایش داده و بدون افزودن اطلاعات مفید، توکن‌ها را مصرف می‌کنند. تیم Vercel به‌روزرسانی‌هایی را در صفحات وبلاگ و changelog خود اعمال کرده است تا markdown را برای عامل‌های هوش مصنوعی قابل دسترس کنند، در حالی که همچنان تجربه کامل HTML و CSS را برای خوانندگان انسانی ارائه می‌دهند. این کار از طریق مکانیسم content negotiation در HTTP انجام می‌شود، جایی که سرور فرمت‌های مختلفی را برای یک محتوا بر اساس درخواست مشتری برمی‌گرداند. این روش از ایجاد محتوای تکراری یا وب‌سایت‌های جداگانه جلوگیری می‌کند. عامل‌های هوش مصنوعی از هدر HTTP "Accept" برای مشخص کردن فرمت‌های ترجیحی خود استفاده می‌کنند. به عنوان مثال، Claude Code هدر زیر را هنگام دریافت صفحات ارسال می‌کند: Accept: text/markdown, text/html, */*. با لیست کردن text/markdown در اولویت، عامل نشان می‌دهد که markdown در صورت وجود، نسبت به HTML ترجیح داده می‌شود. این امکان به توسعه‌دهندگان اجازه می‌دهد تا محتوای بهینه‌شده برای عامل‌ها ارائه دهند و در عین حال سازگاری با کاربران انسانی را حفظ کنند. این به‌روزرسانی مزایای عملکردی قابل توجهی را ارائه می‌دهد. یک پست وبلاگ معمولی با تمام HTML، CSS و جاوااسکریپت می‌تواند 500 کیلوبایت وزن داشته باشد، در حالی که همان محتوا به صورت markdown تنها 2 کیلوبایت است. این یک کاهش 99.6 درصدی در اندازه payload است. برای عامل‌هایی که تحت محدودیت توکن کار می‌کنند، payloadهای کوچکتر به معنای مصرف محتوای بیشتر در هر درخواست و صرفه‌جویی در بودجه برای اطلاعات واقعی به جای markup است.
