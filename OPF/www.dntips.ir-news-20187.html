عنوان:

‫به‌روزرسانی خودکار برنامه‌های دات‌نت با استفاده از فریم‌ورک‌های متن‌باز


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۱/۰۲ ۰۹:۴۶
آدرس: www.dntips.ir
مقدمه
در دنیای توسعه نرم‌افزار، به‌روزرسانی خودکار برنامه‌ها، یکی از نیازهای اساسی برای حفظ کارایی، امنیت و تجربه کاربری مطلوب است. توسعه‌دهندگان Microsoft .NET Framework اغلب با چالش‌هایی مانند مدیریت نسخه‌ها، انتشار سریع به‌روزرسانی‌ها و کاهش بار دانلود برای کاربران مواجه هستند. در این راستا، پروژه‌های متن‌باز متعددی برای خودکارسازی این فرآیند طراحی شده‌اند که هرکدام ویژگی‌ها و رویکردهای خاص خود را دارند. این مقاله به بررسی سه پروژه برجسته در این حوزه، یعنی Velopack، Onova و Squirrel.Windows، می‌پردازد. هدف این است که با تحلیل این ابزارها، نکات کلیدی و مشترک آن‌ها استخراج شده و راهنمایی منسجم برای توسعه‌دهندگان دات‌نت ارائه شود تا بتوانند بهترین ابزار را برای پروژه‌های خود انتخاب کنند.


مفهوم به‌روزرسانی خودکار در برنامه‌های دات‌نت
به‌روزرسانی خودکار (Automatic Updates) فرآیندی است که طی آن برنامه بدون دخالت مستقیم کاربر، نسخه جدید خود را شناسایی، دانلود و نصب می‌کند. این قابلیت به‌ویژه در برنامه‌های دسکتاپ که با دات‌نت نوشته شده‌اند، اهمیت دارد؛ زیرا کاربران انتظار دارند بدون پیچیدگی‌های نصب دستی، همیشه از آخرین نسخه استفاده کنند. سه پروژه مورد بررسی، یعنی Velopack، Onova و Squirrel.Windows، همگی با هدف ساده‌سازی این فرآیند طراحی شده‌اند، اما هر یک رویکرد متفاوتی را دنبال می‌کنند.

معرفی ابزارها و ویژگی‌های کلیدی
  1. Velopack: یک فریم‌ورک مدرن و چندسکویی (Cross-Platform) برای نصب و به‌روزرسانی خودکار برنامه‌ها است که با زبان Rust نوشته شده و عملکردی سریع ارائه می‌دهد. این ابزار از بسته‌های دلتا (Delta Packages) پشتیبانی می‌کند که فقط تغییرات بین نسخه‌ها را دانلود می‌کند و زمان و حجم دانلود را کاهش می‌دهد. Velopack همچنین از زبان‌های برنامه‌نویسی مختلف مانند #C++، C و Rust پشتیبانی می‌کند و مهاجرت از ابزارهای قدیمی‌تر مانند Squirrel را تسهیل می‌کند.
  2. Onova: یک کتابخانه سبک و انعطاف‌پذیر برای به‌روزرسانی برنامه‌های دات‌نت است که بر سادگی و عدم وابستگی به روش نصب خاص تمرکز دارد. Onova به توسعه‌دهندگان امکان می‌دهد تا به‌روزرسانی‌ها را از منابع مختلف مانند وب یا فایل‌های محلی دریافت کنند و از نصب‌کننده‌های مستقل پشتیبانی می‌کند.
  3. Squirrel.Windows: یکی از قدیمی‌ترین ابزارهای این حوزه است که برای برنامه‌های ویندوزی طراحی شده و به دلیل سادگی و ادغام آسان با دات‌نت محبوبیت زیادی کسب کرده بود. با این حال، این پروژه اکنون از رده خارج شده و توسعه فعال آن متوقف شده است، اما همچنان به‌عنوان پایه‌ای برای ابزارهای جدیدتر مانند Velopack مورد استفاده قرار می‌گیرد.

نکات مشترک
  • پشتیبانی از دات‌نت: هر سه ابزار با Microsoft .NET Framework سازگارند و برای توسعه‌دهندگانی که با #C کار می‌کنند، گزینه‌های مناسبی هستند.
  • نصب بدون نیاز به دسترسی ادمین: این ابزارها برنامه‌ها را در پوشه‌های محلی کاربر (Local AppData) نصب می‌کنند و نیازی به مجوزهای سطح administrator ندارند.
  • مدیریت نسخه‌ها: هر سه پروژه امکان بررسی نسخه‌های جدید و اعمال به‌روزرسانی‌ها را فراهم می‌کنند، اگرچه روش‌های پیاده‌سازی آن‌ها متفاوت است.
  • سادگی استفاده: هدف مشترک این ابزارها کاهش پیچیدگی فرآیند انتشار و به‌روزرسانی برای توسعه‌دهندگان است.

تفاوت‌ها و نقاط قوت
  • عملکرد و سرعت: Velopack به لطف استفاده از Rust و بسته‌های دلتا، سریع‌ترین عملکرد را در میان این سه ارائه می‌دهد. Squirrel.Windows و Onova چنین بهینه‌سازی‌هایی ندارند و معمولاً کل بسته را دانلود می‌کنند.
  • پشتیبانی چندسکویی: Velopack از Windows، macOS و Linux پشتیبانی می‌کند، در حالی که Squirrel.Windows فقط برای ویندوز طراحی شده و Onova بیشتر بر انعطاف‌پذیری متمرکز است تا پشتیبانی رسمی چندسکویی.
  • وضعیت توسعه: Squirrel.Windows دیگر به‌روز نمی‌شود، اما Velopack و Onova فعالانه توسعه می‌یابند و به نیازهای مدرن پاسخ می‌دهند.
  • انعطاف‌پذیری: Onova با جدا کردن فرآیند نصب از به‌روزرسانی، انعطاف بیشتری به توسعه‌دهندگان می‌دهد، در حالی که Velopack و Squirrel.Windows نصب و به‌روزرسانی را در یک سیستم یکپارچه ترکیب می‌کنند.

مثال‌های عملی پیاده‌سازی

استفاده از Velopack
Velopack با چند خط کد ساده، به‌روزرسانی خودکار را فعال می‌کند. مثال زیر نحوه استفاده از آن در یک برنامه #C را نشان می‌دهد:
using Velopack;

class Program
{
    static void Main(string[] args)
    {
        VelopackApp.Build().Run(); // راه‌اندازی اولیه
        var source = new HttpSource("https://example.com/updates");
        var um = new UpdateManager(source);
        var updates = um.CheckForUpdates();
        if (updates.HasUpdate)
        {
            um.DownloadUpdates(updates);
            um.ApplyUpdatesAndRestart(updates);
        }
    }
}
این کد ابتدا برنامه را راه‌اندازی می‌کند، سپس بررسی می‌کند که آیا به‌روزرسانی موجود است یا خیر و در صورت وجود، آن را دانلود و اعمال می‌کند.

استفاده از Onova
Onova با استفاده از یک رویکرد ماژولار، به این صورت پیاده‌سازی می‌شود:
using Onova;
using Onova.Services;

class Program
{
    static async Task Main(string[] args)
    {
        using var mgr = new UpdateManager(
            new WebPackageResolver("https://example.com/files/MANIFEST"),
            new ZipPackageExtractor());
        var result = await mgr.CheckForUpdatesAsync();
        if (result.CanUpdate)
        {
            await mgr.PrepareUpdateAsync(result.LastVersion);
            mgr.LaunchUpdater(result.LastVersion);
        }
    }
}
این مثال نشان می‌دهد که Onova چگونه به صورت غیرهمزمان (Asynchronous) به‌روزرسانی‌ها را مدیریت می‌کند.

استفاده از Squirrel.Windows
Squirrel.Windows از رویدادهای خاصی برای مدیریت نصب و به‌روزرسانی استفاده می‌کند:
using Squirrel;

class Program
{
    static void Main(string[] args)
    {
        SquirrelAwareApp.HandleEvents(
            onInitialInstall: (v, tools) => tools.CreateShortcutForThisExe(ShortcutLocation.Desktop),
            onAppUninstall: (v, tools) => tools.RemoveShortcutForThisExe(ShortcutLocation.Desktop),
            onEveryRun: (v, tools, firstRun) => { if (firstRun) Console.WriteLine("نصب شد!"); }
        );
    }
}
این کد نشان‌دهنده مدیریت رویدادهای نصب و حذف در Squirrel است که امروزه توسط Velopack ساده‌تر شده است.

مقایسه
یکی از تفاوت‌های قابل توجه، اندازه بسته‌های به‌روزرسانی است. Velopack با بسته‌های دلتا حجم دانلود را کاهش می‌دهد، اما در مهاجرت از Squirrel، اولین نسخه ممکن است به دلیل تفاوت ساختار پوشه‌ها، اندازه یکسانی برای بسته کامل و دلتا داشته باشد. این مشکل در مستندات Velopack توضیح داده شده و راه‌حل‌هایی برای تنظیم ساختار ارائه شده است. Onova چنین مشکلی ندارد، زیرا به‌روزرسانی را از نصب جدا می‌کند، اما این انعطاف‌پذیری ممکن است پیچیدگی بیشتری به پروژه اضافه کند.

نتیجه‌گیری
برای توسعه‌دهندگان دات‌نت که به دنبال راه‌حلی مدرن و سریع هستند، Velopack با پشتیبانی چندسکویی و عملکرد بهینه، گزینه‌ای ایده‌آل است. Onova برای پروژه‌هایی که نیاز به انعطاف‌پذیری بالا دارند مناسب است، در حالی که Squirrel.Windows فقط برای برنامه‌های قدیمی‌تر که هنوز از آن استفاده می‌کنند، قابل توصیه است. انتخاب ابزار مناسب به نیازهای پروژه، مانند سرعت، مقیاس‌پذیری و سادگی، بستگی دارد. پیشنهاد می‌شود توسعه‌دهندگان با آزمایش این ابزارها در پروژه‌های کوچک، بهترین گزینه را برای خود پیدا کنند.
