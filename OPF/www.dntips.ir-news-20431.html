‫تقویت گردش کار VS Code با GitHub MCP و Copilot Agent


نویسنده: وحید نصیری
تاریخ: ۱۴۰۴/۰۲/۳۰ ۰۷:۴۶
آدرس: www.dntips.ir
این مقاله به بررسی روش‌های افزایش چشمگیر کارایی گردش کار در Visual Studio Code (VS Code) از طریق یکپارچه‌سازی پروتکل متن مدل گیت‌هاب (GitHub Model Context Protocol) یا به اختصار MCP، با حالت Agent کُوپایلوت (Copilot Agent Mode) می‌پردازد. این ترکیب قدرتمند به توسعه‌دهندگان امکان می‌دهد تا با استفاده از زبان طبیعی و دستورات صوتی، وظایف مربوط به کد و مخازن گیت‌هاب را به صورت خودکار انجام دهند. این مقاله با تمرکز بر مخاطبان .NET، به تشریح مفاهیم کلیدی، مراحل راه‌اندازی، قابلیت‌ها و مزایای این رویکرد نوین می‌پردازد.

پروتکل متن مدل گیت‌هاب (GitHub MCP):
یک سرور است که به عنوان واسط بین کُوپایلوت و گیت‌هاب عمل می‌کند و امکان اجرای خودکار دستورات گیت‌هاب را بر اساس درخواست‌های زبان طبیعی فراهم می‌سازد. به عبارت دیگر، MCP به کُوپایلوت اجازه می‌دهد تا با گیت‌هاب تعامل داشته باشد و عملیاتی مانند ایجاد مخزن، تغییر فایل‌ها، کامیت (Commit) و پوش (Push) کردن کد را به صورت خودکار انجام دهد. اطلاعات بیشتر در مورد MCP

حالت Agent کُوپایلوت (Copilot Agent Mode): یک افزونه برای گیت‌هاب کُوپایلوت است که به آن امکان می‌دهد تا به جای صرفاً پیشنهاد دادن کد، به طور مستقیم اقداماتی را انجام دهد. این حالت، کُوپایلوت را به یک دستیار کدنویسی تبدیل می‌کند که می‌تواند وظایف را به صورت خودکار اجرا کند. اطلاعات بیشتر در مورد Copilot Agent

مراحل راه‌اندازی
برای بهره‌مندی از این ترکیب قدرتمند، مراحل زیر باید انجام شوند:
فعال‌سازی حالت Agent در VS Code: در تنظیمات VS Code، گزینه "Agent" را فعال کنید.


فعال‌سازی یکپارچه‌سازی GitHub MCP: در تنظیمات VS Code، به دنبال "MCP" بگردید و گزینه "Edit in settings.json" را انتخاب کنید.


پیکربندی GitHub MCP در settings.json: یک پیکربندی "mcp" اضافه کنید که دستور اجرای سرور GitHub MCP و یک متغیر محیطی برای توکن دسترسی شخصی گیت‌هاب (GitHub personal access token) را مشخص می‌کند. نمونه‌ای از این پیکربندی در زیر آمده است:
JSON

"mcp": {
    
        "inputs": [],
        "servers": {
            "github": {
                "command": "npx",
                "args": [
                    "-y",
                    "@modelcontextprotocol/server-github"
                ],
                "env": {
                    "GITHUB_PERSONAL_ACCESS_TOKEN": "YOUR_GITHUB_ACCESS_TOKEN"
                },
            }
        }
    }

دریافت توکن دسترسی گیت‌هاب: یک توکن "Fine-grained token" در گیت‌هاب با دسترسی‌ها و مجوزهای لازم برای مخازن خود ایجاد کنید. آموزش ساخت توکن دسترسی


قابلیت‌ها
با انجام تنظیمات فوق، کاربران قادر خواهند بود تا:
  • ایجاد مخازن: با استفاده از دستورات زبان طبیعی، مخازن جدید گیت‌هاب ایجاد کنند.
  • تولید کد: توصیف عملکردهای مورد نیاز در زبان طبیعی و تولید کد توسط کُوپایلوت.
  • استفاده از دستورات صوتی: انجام اقدامات مختلف با استفاده از ورودی صوتی، مانند افزودن توضیحات.
  • ایجاد مستندات: تولید فایل‌های README و سایر مستندات پروژه با استفاده از زبان طبیعی.
  • مدیریت Git: کامیت و پوش کردن تغییرات به گیت‌هاب با استفاده از دستورات ساده متنی یا صوتی.


مزایا
استفاده از این ترکیب مزایای متعددی را برای توسعه‌دهندگان به همراه دارد، از جمله:
  • کاهش تغییر زمینه (Context Switching): بیشتر وظایف مربوط به Git و مخزن را در VS Code انجام دهید.
  • کاهش بار شناختی: نیاز به یادآوری دستورات خاص Git را از بین ببرید.
  • نمونه‌سازی سریع‌تر: به سرعت مخازن ایجاد کرده و پروژه‌ها را آغاز کنید.
  • مستندسازی بهتر: به راحتی مستندات پروژه را تولید کنید.
  • افزایش دسترسی‌پذیری: تعامل با Git را برای توسعه‌دهندگان جدید آسان‌تر کنید.

نتیجه‌گیری
یکپارچه‌سازی GitHub MCP و Copilot Agent نشان‌دهنده یک تغییر اساسی به سوی گردش کارهای توسعه‌ای کارآمدتر و مکالمه‌ای است. این رویکرد به توسعه‌دهندگان امکان می‌دهد تا به جای درگیر شدن با جزئیات فنی و دستورات پیچیده، بر حل مسائل تمرکز کنند. با استفاده از این ابزارها، توسعه‌دهندگان .NET می‌توانند سرعت و کیفیت کار خود را به طور قابل توجهی افزایش دهند.
